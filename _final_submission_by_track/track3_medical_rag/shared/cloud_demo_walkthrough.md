# ControlMind 云端 Demo 功能导览

> 本文件定位：说明云端 Demo 如何承载三赛道公开体验、来源回放与边界展示。云端 Demo 是完整系统的公开交互切面，不替代本地私有执行、长任务复现和原始数据审计。

## 访问入口

- 云端 Demo：https://demo.askiler.com/
- HTTP 入口：http://demo.askiler.com/（会自动跳转到 HTTPS）
- 前端访问码：`ControlMind@2026`

打开页面后，在顶部状态栏的“访问码”输入框填入上述访问码并点击“保存”。该访问码只用于解锁公开云端实时调用，不是服务器后台密码；公开页面仍只处理公开 PDF、公开 URL 或脱敏问题。

## 总览页

打开 `/home`。页面首先呈现三赛道卡片、公开展示边界和实时云端依赖。

云端 Demo 不是完整系统的替代品。它保留公开 PDF 解析、出题评分、任务规划和医学 RAG 来源回放；私有语料、索引、模型权重和批量实验留在本地完整系统内复核。

## 演示路径页

打开 `/demo`。页面提供一条从总览、赛道样例到证据核验的推荐路径。

“后端 API 闭环”和“验收核验”体现了云端切面的设计原则：能公开执行的动作保留实时入口，长任务展示已验证产物回放。云端页面与完整工作台保持同一套边界，不展示私有资产可用性，也不把回放伪装成实时全量执行。

## 赛道一：公开论文解析、出题与评分

打开 `/track1`。先看语料规模、方法论和边界说明，再用公开 PDF 上传或公开 URL 走 MinerU 官方 API。解析完成后点击“云端出题”，再点击“云端评分”。

这一部分展示赛道一的公开闭环能力。公开输入可以真实进入 MinerU 解析和 DeepSeek 出题评分；题目、参考答案、评分理由和来源可以在同一页面核验。全量 500 题榜单不在云端实时重跑，其结果通过本地 CLI、DATA-TRACE 和报告附录复核。

## 赛道二：Data Agent 产物回放

打开 `/track2`。选择一个公开任务模板，查看 intent、DAG、资源策略和来源产物。

这一部分展示赛道二的执行协议层能力。语料生产被拆成 intent、DAG、资源策略、工具调用和来源产物；云端展示短路径规划和已验证日志，长任务、私有资源调度和批量飞轮不在公网重跑。

## 赛道三：医学 RAG 来源回放

打开 `/track3`。点击“闭环胰岛素机制解释”，再看回答、来源卡片、claim 状态和安全边界。

这一部分展示医学 RAG 的循证问答边界。系统先检索，再组织来源，再校验结论，最后暴露安全边界。回答可以解释机制，例如 48 个月内胰岛素需求上升为什么使自适应闭环有意义，同时明确不替代个体治疗、处方或急诊判断。

## 来源矩阵

打开 `/evidence`。按三赛道查看报告、数据、API 和验收路径之间的对应关系。

这一页面连接报告、数据、API 与验收路径。云端展示内容可以回到 DATA-TRACE、trace JSONL、CLI 和样例包。

## 云端状态与边界页

打开 `/health` 和 `/boundary`。先看公开云端状态，再看云端 Demo 与完整系统的边界表。

云端等于公开/脱敏输入、服务端代理、trace 回放；完整系统等于私有资产、长任务、索引/模型和批量实验。ControlMind 的价值不是把所有东西搬上公网，而是在公开体验和私有部署之间保持同一套方法论。

## 能力边界

- 云端 Demo 不承载全量重跑、真实长任务、本地模型队列或私有索引。
- 患者材料、私有文档、chunk、索引、模型权重和批量实验日志不进入公网展示。
- 医学回答仅作为文献证据综合与机制解释，不构成诊疗、处方或急诊建议。
- trace 回放不是静态装饰页面，而是完整系统中已验证链路的公开镜像。
