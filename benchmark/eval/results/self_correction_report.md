# §5.6 Self-Correction 实验：Feedback-Driven 修正作答

> 生成时间: 2026-05-07 00:35:14 | 修正模型: deepseek-v4-flash | Judge: deepseek-v4-flash | 并发: 4 线程 | 耗时: 289.88s

## 5.6.1 实验设计

从 DeepSeek-v4-flash 的 500 题评测结果中，筛选低分题（score < 0.3），按 A/B/C/D 四维各取 5 题（seed=42，共 20 题）。每题构造 feedback prompt：

> 你之前的回答得分 {score}，扣分理由：{reason}
> 请基于此反馈重新作答以下问题：{question}

用同一 DeepSeek-v4-flash 调用修正 API 获取修正答案，再用同一 Judge（DeepSeek-v4-flash）按原维度评分规则对修正答案评分，记录原始分数、修正分数与 Δ。

### 四维修正成功率分布

| 维度 | 题数 | Δ>0 | 成功率 | 原始均分 | 修正均分 | 平均 Δ |
|------|:---:|:---:|:-----:|:--------:|:--------:|:------:|
| 概念回溯 (A) | 5 | 4 | 80% | 0.00 | 0.72 | +0.72 |
| 多步推理链 (B) | 5 | 1 | 20% | 0.12 | 0.24 | +0.12 |

**分析**：A 维（概念回溯）修正效果最好（4/5 有效），因为扣分理由多为「核心概念不一致」「答非所问」——feedback 直接指出了问题所在。B 维（多步推理链）修正效果最差（1/5），因为数学推导链一旦断裂，单条 feedback 难以重建。D 维修正后均分提升显著（0.18→0.45），其中 CS-EVO-00630 从 0.24 跃升至 1.0。

## 5.6.4 结论与 v2 路线图

### 结论

- **Self-Correction 对概念型错误高度有效**：A 维 80% 的修正成功率表明，当错误是「理解偏差」而非「能力不足」时，Judge feedback 可以精准纠偏。
- **数学推理修正困难**：B 维仅 1/5 有效，多步数学推导链断裂后，单轮 feedback 很难重建完整推理。
- **D 维开放设计有提升空间**：部分 D 维题修正后分数大幅提升（CS-EVO-00630: 0.24→1.0），说明设计类问题可以在 feedback 引导下改进，但依赖反馈的具体性。
- **总体修正率 50%**：10/20 题 Δ>0，修正后均分从 0.10 提升至 0.45，提升约 4.5 倍。但负面案例表明单一 feedback 轮次不足以修复结构性问题。

### v2 路线图：Pipeline JSON 格式 + Tool-Calling 标注

当前 self-correction 实验存在两个局限，规划 v2 改进：

**1. Pipeline JSON 格式**

当前数据集为扁平 QA 对格式（question + answer），不利于结构化推理链的 self-correction。v2 规划：
- 每条题目以 pipeline JSON 格式存储，包含多步推理步骤（`reasoning_steps`）、关键中间变量、验证检查点。
- Self-correction 可定位到具体的错误步骤（而非仅题目级别的 feedback），提修正精度。
- 参考 `self_correction_trajectory.json` 中的 20 条完整轨迹作为 v2 格式设计基准。

**2. Tool-Calling 标注**

- 对需要数值计算、矩阵求解、仿真验证的题目（B/D 维高频），标注工具调用需求（如 `compute_eigenvalues`、`solve_riccati`、`simulate_step_response`）。
- Self-correction 流程中：**模型输出 → Tool 调用验证 → 验证结果反馈 → 模型修正**，形成闭环。
- 工具调用结果作为结构化 evidence 注入 feedback prompt，替代当前纯文本 Judge 理由。

> **注**：结构化研究链 JSON 和工具调用数据不纳入本次赛道一交付，作为 v2 路线图声明。
