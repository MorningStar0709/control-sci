## 一、问题定义

**PPL 悖论 (Perplexity Paradox)**：QLoRA 微调使模型对领域文本的预测困惑度大幅下降（信息论角度证明领域适应有效），但 Judge 评分几乎不变甚至轻微退化（实证角度证明答题能力未提升）。两者之间的量化差距即是悖论的核心度量。

### 量化框架

```
悖论度量 = PPL Δ% 与 Judge Δ% 的差距
         = 信息论改善 vs 实证退化/不变
         
悖论系数 = |PPL Δ%| / |Judge Δ%|
         = 信息论改善幅度是实证变化的多少倍
```

| 对比维度 | PPL Δ% | Judge Δ% | 差距 | 悖论系数 |

| PPL 改善 vs Judge 退化（4B vs 4B基线） | **-53.5%** | **-0.73%** | 52.8pp | **73x** |

> **核心悖论**：PPL 改善 53.5% 意味着模型对控制科学领域答案的语言分布拟合度大幅提升（信息论角度），但 Judge 评分仅退化 0.73%（实证角度）。信息论改善是实证退化的 **73 倍**。
>
> PPL 改善并不自动转化为更好的答题质量——这是领域微调中最危险的认识误区之一。

## 五、悖论机制分析

### 5.1 为什么 PPL 改善 ≠ Judge 提升？

| 机制 | 解释 | 证据 |
|:----|:-----|:-----|
| **语言分布 vs 推理质量** | PPL 衡量对标准答案 token 序列的预测概率；Judge 衡量答案的推理正确性、完整性和格式。PPL 改善说明语言分布拟合更好，但不保证推理质量提升 | 4B PPL -53.5% vs Judge -0.73% |
| **C 维精确幸存** | C 维（条件敏感性）在 QLoRA 后 Δ=+0.0000，但全量 89 题 PPL 均下降。PPL 是 corpus 级整体指标，无法拆分到维度；C 维 Judge Δ=0 只是独立地说明——即使 PPL 中包含 C 维题目的信息利得，这种利得也未转化为 C 维 Judge 评分的提升 | C 维 Δ=0, PPL 89/89 改善 |
| **LoRA 容量约束** | r=16 的 LoRA 适配器主要注入词汇/句型层面的领域知识（降低 PPL），对深层推理能力的改进有限 | 两尺寸 PPL 收敛到同一水平 |
| **Judge 的格式敏感性** | QLoRA 输出格式可能略有变化，Judge 可能因格式扣分抵消了内容改善 | 部分维度的 issues 统计待补充 |

### 5.2 PPL 的合理使用边界

| 适合 | 不适合 |
|:----|:-------|
| 验证领域适应是否有效（是否需要重训） | 预测端到端 Judge 评分 |
| 比较不同适配器/超参的信息吸收效率 | 替代 end-to-end 评测 |
| 早期快速筛选用（3min vs 70min） | 作为唯一的评测指标 |
| 跨尺寸公平对比（不受 Judge 一致性影响） | 诊断各维度能力变化 |

### 5.3 悖论的工程含义

1. **PPL 不能作为 QLoRA 质量代理指标** — PPL 改善不保证答题质量提升
2. **PPL 探针是必要条件，不是充分条件** — PPL 下降可证领域适应有效，但无法替代 end-to-end 评测
3. **C 维精确零变化的反直觉含义**：dense language knowledge injection ≠ per-dimension capability improvement
4. **推荐的最佳实践**：PPL 探针（3min）+ end-to-end Judge 评测（70min）双路径验证，缺失任一都会系统性地误导结论

## 六、数据质量与限制

| 限制项 | 影响 |
|:------|:-----|
| PPL 不分维度 | 无法定位 PPL 改善来自哪个维度 |
| 9B QLoRA 无 Judge 评分 | 9B 悖论无法直接验证（仅推断） |
| 4B baseline 是 Ollama qwen3.5:4b（instruct），与 4B text-only 基座架构不同 | PPL 和 Judge 的基座不同，但二者在各自维度内可比 |
| PPL 探针计算 loss 的 prompt 结构（system+user+assistant 全序列）与 Judge 评分 prompt 不同 | 但 system/user tokens 在 base 和 QLoRA 间相同，ΔPPL 不受影响 |
