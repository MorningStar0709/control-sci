# 评分者悖论：模型尺寸与评分能力的反向关系

> **独立挑战文档 (§5)** | 生成日期：2026-05-09
> **核心发现**：在跨模态对齐评测中，小模型（gemma3:4b, 4.4GB）以 0.872 评分表现完胜大模型（qwen3.5:35b, 23GB）的 0.233——尺寸与评分能力呈显著反比。
> **数据来源**：本地 6 Judge × 80 题抽检 + API 8 Judge × 500 题全量，双路径交叉验证

---

## 1. 发现概述

### 研究追问链

```
赛道一：9 模型答题评测 → 发现"模型尺寸与答题质量不相关"
  （小模型如 MiMo-v2-flash 夺冠，大模型如 MiMo-v2.5 垫底）
    │
    ▼
赛道二追问：评分者角色是否有同样现象？
    │
    ├─ 本地 6 Judge × 80 题 ✅ — 已证实：尺寸与评分能力呈反比！
    │   gemma3:4b(4.4GB) 0.872 > llama3.2:3b(2.0GB) 0.678 > qwen3.5:35b(23GB) 0.233
    │   Fleiss' κ = 0.003（各模型评分体系完全独立）
    │
    └─ API 8 Judge × 500 题 ✅ — API 端同样存在尺寸≠评分能力的现象
        mimo-v2-pro（API 最大/最贵）0.7091 🥇 > deepseek-v4-pro（大）0.6018
```

### 核心结论

**评分者悖论 (Rater Paradox)**：在科学文档跨模态对齐评分任务中，模型担任评分者角色时，其评分严格度与模型尺寸呈反比——较小模型倾向于给出更高的评分，而较大模型反而评分偏低。这一现象在本地 Ollama 端表现为**极端反比**（最大模型 35B 23GB 评分最低 0.233，最高分评分者 gemma3:4b 4.4GB 仅中等大小），在 API 端以弱化形式存在。

## 3. 本地端 Judge 矩阵：尺寸与评分的反比关系

### 3.1 总览

| 排名 | 模型 | 尺寸 | 磁盘大小 | overall | 耗时(全量预估) |
|:--:|------|:--:|:--:|:--:|:--:|
| 🥇 | **gemma3:4b** | 小 | 4.4GB | **0.872** | 2.4min |
| 🥈 | llama3.2:3b | 极小 | 2.0GB | **0.678** | 2.4min |
| 🥉 | qwen3.5:2b | 极小 | 1.6GB | 0.568 | 0.6min |
| 4 | qwen3.5:9b | 中 | 5.5GB | 0.462 | 1.1min |
| 5 | qwen3.5:4b | 小 | 2.5GB | 0.347 | 1.4min |
| 6 | qwen3.5:35b | 大 | 23GB | 0.233 | 1.9min |

### 3.2 关键发现

1. **极端反比**：最大模型（qwen3.5:35b, 23GB）评分最低（0.233），而评分最高的模型（gemma3:4b, 4.4GB）为中等大小，不到 35B 磁盘体积的 1/5。6 个模型按磁盘大小与评分的 Pearson r ≈ -0.60（中等负相关）。极端值趋势鲜明，但非严格单调——qwen3.5:4b (0.347) 低于 qwen3.5:9b (0.462)，gemma3:4b (4.4GB, 0.872) 高于更小的 llama3.2:3b (2.0GB, 0.678)。
2. **跨系列泛化**：此现象跨模型系列成立——Qwen 系列内部（2b 0.568 > 4b 0.347 > 9b 0.462 > 35b 0.233）、Gemma（0.872）和 Llama（0.678）均呈现"小模型更友好，大模型更严苛"的趋势。
3. **异常点注释**：qwen3.5:4b (0.347) 略低于 9b (0.462)，打破 Qwen 内部严格单调递减。可能原因：4B 参数量在该架构下处于"能力洼地"——不够大以建立稳定的评分准则，又不够小以维持简单一致的判断倾向。

### 3.3 各维度评分分解

|:--:|------|:--:|:--:|:--:|:--:|:--:|

| 6 | MiniMax-M2.5 | MiniMax | 中 | 0.6370 | 14 | 2.8% |
| 7 | deepseek-v4-flash | DeepSeek | 小 | 0.6321 | 5 | 1.0% |
| 8 | deepseek-v4-pro | DeepSeek | 大 | 0.6018 | 8 | 1.6% |

### 4.2 API 端 vs 本地端对比

| 维度 | 本地端 | API 端 |
|:--|:--|:--|
| 趋势 | 尺寸↑ → 评分↓ 极端反比 | 尺寸↑ → 评分↓ 弱相关 |
| 最高分评分者 | gemma3:4b (0.872) | mimo-v2-pro (0.709) |
| 最低分评分者 | qwen3.5:35b (0.233) | deepseek-v4-pro (0.602) |
| 评分跨度 | 0.639 | 0.107 |
| 一致性 (Fleiss' κ) | 0.003（几乎零） | 0.575（中等） |

### 4.3 跨 Provider 模式

- **MiMo 系列**：4 个模型间跨度最大（0.652-0.709, Δ=0.057），mimo-v2-pro（最大/最贵）反而评分最高——这是 API 端唯一"大模型更高分"的例外，但该模型在答题评测中排名第 8（0.514），属于弱答题者。
- **DeepSeek 系列**：v4-flash（小）> v4-pro（大），符合悖论趋势，但差距仅 0.030。
- **MiniMax 系列**：M2.7（大）> M2.5（中），与悖论趋势相悖，差距 0.012（可视为噪声）。

**API 端核心结论**：评分者悖论在 API 端以弱化形式存在。API 模型的评分差异远小于本地端（跨度 0.107 vs 0.639），且受 Provider 系列内部一致性约束。mimo-v2-pro 作为答题最弱模型（0.514）却是最高分评分者（0.709），构成悖论的子命题——"答题能力与评分严格度不相关"。

## 5. 评分者一致性分析

### 5.1 本地端：Fleiss' κ = 0.003

6 个本地模型 × 80 题的 Fleiss' κ 接近零，意味着各模型的评分体系**完全独立**——它们评估同一道题时用的根本是不同标准。这与评分均值差异巨大（0.233-0.872）一致：不仅是"松紧度"不同，而是对"什么是对的"有本质性分歧。

**Cohen's κ 成对矩阵（最高仅 0.38）：**

| 模型对 | Cohen's κ | 解释 |
|------|:--:|:--|
| qwen3.5:4b ↔ qwen3.5:9b | 0.38 | 同系列弱一致 |
| qwen3.5:9b ↔ qwen3.5:35b | 0.31 | 同系列弱一致 |
| gemma3:4b ↔ 任何其他 | < 0.10 | 完全独立体系 |
| llama3.2:3b ↔ 任何其他 | < 0.15 | 完全独立体系 |

### 5.2 API 端：Fleiss' κ = 0.575

3 家 API Judge（deepseek-v4-flash, deepseek-v4-pro, MiniMax-M2.5）在 27/30 题上的 Fleiss' κ = 0.575，属于 **moderate agreement**（0.41-0.60）。观测一致率 P_bar = 0.827，随机期望 P_e_bar = 0.593。

### 5.3 双端对比：本地 vs API

| 指标 | 本地 6 Judge | API 3 Judge (基线) | API 8 Judge |
|:--|:--:|:--:|:--:|
| Fleiss' κ | 0.003 | 0.575 | 待计算 |
| 评分均值范围 | 0.233-0.872 | 0.683-0.720 | 0.602-0.709 |
| 评分标准差范围 | 大 | 0.358-0.419 | — |
| 排除题数 | 0/80 | 3/30（解析异常） | — |

**启示**：本地端各模型几乎"各评各的"，不能做跨模型平均或投票。API 端存在中等一致性，三家以上 API 的投票/平均可作为合理基线。

## 6. 跨赛道串联：从答题到评分的统一悖论

### 6.1 赛道一：尺寸与答题质量不相关

| 排名 | 模型 | 答题 overall | 尺寸分类 |
|:--:|------|:--:|:--:|
| 🥇 | MiMo-v2-flash | 0.647 | 小 |
| 🥈 | deepseek-v4-flash | 0.632 | 小 |
| 🥉 | qwen3.5:9b | 0.625 | 中 |
| ... | ... | ... | ... |
| 9 | MiMo-v2.5 | 0.440 | 小 |

核心发现：小模型登顶，大模型未占优。答题能力与模型尺寸、成本均无单调关系。

### 6.2 赛道二：尺寸与评分能力呈反比

| 排名 | Judge | Judge overall | 尺寸分类 |
|:--:|------|:--:|:--:|
| 🥇 | gemma3:4b | 0.872 | **小** |
| 6 | qwen3.5:35b | 0.233 | **最大** |

核心发现：担任评分者角色时，小模型更"友好宽松"，大模型更"严苛保守"。趋势比答题端更鲜明。

### 6.3 统一悖论框架

```
              答题能力 (赛道一)              评分严格度 (赛道二)
                    │                            │
    小模型 ─────────┤  不相关 ──────────── 显著反比
    大模型 ─────────┤  不相关 ──────────── 显著反比
                    │                            │
                无单调关系                  极端反比
                                           (35B 最低 vs gemma 最高)
```

**三个层次的退化**：
1. **模型越大 ≠ 答题越好**（赛道一，已证实）
2. **模型越大 ≠ 评分越高**（赛道二本地端，已证实）
3. **答题能力与评分严苛度无单调关系**（mimo-v2-pro 答题垫底 0.514 却判分最宽松 0.709，已证实）

## 7. 方法论启示与实用建议

### 7.1 对多 Judge 评测设计的指导

| 建议 | 依据 |
|:--|:--|
| **不要假设"大 Judge 更准"** | 35B 评分 0.233 vs Gemma 0.872，尺寸与评分严格度/质量无单调关系 |
| **本地 Judge 不可互相替代** | Fleiss' κ = 0.003，各模型评分体系完全独立 |
| **API Judge ≥ 3 家可投票/平均** | API Fleiss' κ = 0.575，存在中等一致性 |
| **评分者多样性优于单一大模型** | 多样性覆盖不同评分偏差，降低系统性偏斜风险 |

### 7.2 对 Agent 系统设计的指导

- `multi_judge_compare` intent 应默认选 1 个 API + 1 个本地小模型（如 gemma3:4b），而非单一大模型
- 跨模型评分做分布报告而非均值：报告每个 Judge 的 full distribution，标注离散度
- Agent 的 quality_arbitrate（intent 5）应使用 API 基线 Judge，因其一致性更高

### 7.3 理论假设

评分者悖论的可能机制（待进一步验证）：

1. **大模型的过度批判倾向**：35B 具有更强的模式匹配能力，能识别答案中微小的不完美处，因而系统性低估
2. **小模型的锚定效应**：2B/3B 能力有限，只能匹配表面关键词，倾向于给"看起来合理"的答案高分
3. **评分者的"元认知"差异**：大模型对自身评分的不确定性更高，因此更保守（给出中间值而非极端值）
4. **知识覆盖宽度**：大模型知道更多"可能出错的方式"，导致对每道题都找到扣分理由
