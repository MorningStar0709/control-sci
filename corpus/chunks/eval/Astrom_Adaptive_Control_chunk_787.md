# Block Diagram Representation

The block diagram in Fig. 13.2 represents a solution to a generic signal processing problem. To make it easy to build large systems, it is convenient to consider this module as a building block that can be used for many different purposes. This is simpler if a proper representation is used. For that purpose it is convenient to represent the module as a block that receives signals x and y and delivers estimates $\hat{y}$ and $\hat{\theta}$ . Such a representation, shown in Fig. 13.2(b), makes it possible to describe several adaptive signal processing problems.
