To improve the accuracy of the data-driven representation and the tracking performance, we partition the state space into different regions and construct a corresponding data-driven model for each of them. This can be viewed as a local linear approximation of the nonlinear system. Specifically, as the nonlinearity of the system mainly arises from the coupled terms sin(zδ) · v and $\cos ( z _ { \delta } ) \cdot v$ in states $z _ { x }$ and $z _ { y } ,$ we divide the state-space into 4 regions according to the orientation, that are $[ 0 , \frac { \pi } { 2 } ) , [ \frac { \pi } { 2 } , \pi ) , [ \pi , \frac { 3 \pi } { 2 } )$ and $\textstyle \left[ { \frac { 3 \pi } { 2 } } , 2 \pi \right)$ . We collect 4 trajectories of length 1200 with initial states $( 0 , 0 , \frac { \pi } { 4 } ) , ( 0 , \check { 0 } , \frac { 3 \pi } { 4 } ) , ( 0 , 0 , \frac { 5 \pi } { 4 } )$ and $( 0 , 0 , \frac { 7 \pi } { 4 } )$ to capture the behavior of the nonlinear system within these regions. The input signals v and w are sampled uniformly from [10, 20] and $[ - \frac { \pi } { 6 } , \frac { \pi } { 6 } ]$ , respectively. Then, for each approach, we construct 4 corresponding trajectory libraries over these regions with $\dot { T } _ { \mathrm { i n i } } = 5 ^ { \circ }$ and $N = 1 2$ . During the online tracking, we switch the data library based on the robot’s current orientation.

The closed-loop tracking performance of different approaches is shown in Fig. C.1. All approaches are able to approximately track the prescribable reference trajectory. Among them, the L-DDPC and SPC exhibit noticeable deviation and oscillation behavior, while the C-DDPC and O-DDPC achieve superior tracking performance.
