We briefly describe now the B-RRT algorithm.
The tree $\mathcal { G } = ( \nu , \mathcal { E } )$ to be constructed is initialized in line 2, with the node set  initialized to the system’s initial configuration $q ( 0 )$ , and the respective edge set $\mathcal { E }$ to the empty set.
The algorithm samples then a point $q _ { \mathrm { r a n d } }$ in the extended free space $\bar { \lambda } _ { \mathrm { f r e e } } \big ( \big ( \bar { e } _ { t r } , \bar { \eta } _ { r } \big ) \big )$ .
Then the nearest neighbor $q _ { \mathrm { n e a r e s t } }$ , in terms of $d _ { \mathbb { T } }$ , is found in the tree (line $7 )$ , and a new point $q _ { \mathrm { n e w } }$ on the line between qnearest and $q _ { \mathrm { r a n d } }$ is computed;
$q _ { \mathrm { n e w } }$ can be chosen such that $d _ { \mathbb { T } } ( q _ { \mathrm { n e a r e s t } } , q _ { \mathrm { n e w } } ) = \epsilon$ , according to a predefined incremental distance $\epsilon \ : ( [ 3 6 8 ] )$ .
If the line segment between $q _ { \mathrm { r a n d } }$ and $q _ { \mathrm { n e w } }$ belongs to the extended free space $\bar { \lambda } _ { \mathrm { f r e e } } \big ( ( \bar { e } _ { t r } , \bar { \eta } _ { r } ) \big )$ , then the respective node $q _ { \mathrm { n e w } }$ and edge $\{ q _ { \mathrm { n e a r e s t } } , q _ { \mathrm { n e w } } \}$ are added to the tree (lines 9-11).
After the execution of the algorithm, a standard search algorithm can be employed to find the sequence of edges that lead from $q ( 0 )$ to $Q _ { g }$ .
Moreover, note that the bounds in (6.9) concern (at least twice) continuously differentiable trajectories.
Therefore, the resulting solution path, which is formed by the concatenation of the respective edges in the tree, has to be converted to a such a trajectory.
This procedure might modify the initial path that was checked for collisions, and hence the smooth version should be re-checked for collisions in $\bar { \lambda } _ { \mathrm { f r e e } } \big ( \big ( \bar { e } _ { t r } , \bar { \eta } _ { r } \big ) \big )$ .
Subsequently, the resulting smooth (at least twice cont.
different.) path is endowed with time constraints to derive a timed trajectory $q _ { \mathrm { d } } : [ 0 , t _ { f } ]  \bar { \mathcal { A } } _ { \mathrm { f r e e } } ( ( \bar { e } _ { t r } , \bar { \eta } _ { r } ) )$ , for some $t _ { f } > 0$ , which is given as the desired trajectory input to the control protocol designed in the previous section.
The actual trajectory of the system $q ( t )$ is guaranteed to track $q _ { \mathrm { d } } ( t )$ in the funnel defined by $\bar { e } _ { t r } , \bar { \eta } _ { r }$ .
