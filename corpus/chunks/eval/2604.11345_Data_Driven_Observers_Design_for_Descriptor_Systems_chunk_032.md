Since Assumption 1 holds, the necessary and sufficient condition for the existence of a model-based ESO (or similar variants) is the detectability of the augmented system (61), $\operatorname { i . e . , } r k \left[ { \overset { \lambda E - A } { \boldsymbol { C } } } \right] = n + r , \ \forall \lambda \in \mathbb { C } , \ | \lambda | \geq 1 .$ . This condition can be reformulated by expressing the matrices of the augmented system in terms of the original system matrices $( A _ { 0 } , E _ { 0 } , \dot { C _ { 0 } } , F _ { 0 } )$ . Specifically, the rank condition becomes equivalent to

$$
r k \left[ \begin{array}{c c} \lambda I _ {n} - A _ {0} & - E _ {0} \\ C _ {0} & F _ {0} \end{array} \right] = n + r, \forall \lambda \in \mathbb {C}, | \lambda | \geq 1, \lambda \text {finite}.
$$

This reformulation explicitly links the detectability of the augmented system (61) to the strong detectability of the original system (60) (the triple $( A _ { 0 } ^ { - } , E _ { 0 } , C _ { 0 } , F _ { 0 } )$ is strongly detectable if and only if the above rank condition holds). This model-based existence condition coincides with another augmented system based ESO design method [32], which uses the difference between the future disturbance and the current one, i.e., d(k+1)−d(k), as an external input.

Our proposed method provides a data-driven design of ESO for system (60). The data-driven construction of an ESO fundamentally relies on obtaining input-outputstate data sequences that can sufficiently excite the dynamic modes of system (61). As analyzed in Section $^ { 3 , }$ $\{ u _ { d } ( k ) \} _ { k = 0 } ^ { T + s - 1 }$ o be constructed, its historical input data must first satisfy the persistent excitadesign further requires the historical input-output data $( \{ u _ { d } ( k ) \} _ { k = 0 } ^ { T - 1 } , \{ y _ { d } ( \bar { k } ) \} _ { k = 0 } ^ { T - 1 } )$ , and the augmented state vector $\{ \tilde { x } _ { d } ( k ) \} _ { k = 0 } ^ { T - 1 }$ , which is defined by concatenating the original system state and the disturbance input. Note that without the disturbance input data, it is impossible to estimate $d ( k )$ online in the observer framework, since any invertible coordinate transformation does not alter the system input and output.
