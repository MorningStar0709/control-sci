# B. Least-Squares Policy Iteration

In LSPI, we perform policy evaluation steps called LSTD-Q, followed by policy improvement steps based on the updated Q-function. The Q-function parametrization in LSPI is linear in the parameters, i.e.,

$$\tilde {Q} ^ {L S P I} (x, u, \theta) = \psi^ {\top} (x, u) \theta^ {L S P I}, \tag {21}$$

with θLSP I $\theta ^ { L S P I } \qquad \in \qquad \mathbb { R } ^ { n _ { \theta } } ,$ , and ψ(x, u) := $\left[ \psi _ { 1 } ( x , u ) , \quad \psi _ { 2 } ( x , u ) , \quad \ldots , \quad \psi _ { n _ { \theta } } ( x , u ) \right] ^ { \top }$ a vector of basis functions, with $\psi _ { i } : \mathbb { R } ^ { n _ { x } } \times \mathbb { R } ^ { n _ { u } }  \mathbb { R } , i \in \lbrace 1 , 2 , \ldots , n _ { \theta } \rbrace$ . Note that this Q-function parametrization slightly generalizes (10). The linearity of $\tilde { Q } ^ { L S P I }$ in $\theta ^ { L S P \overline { { I } } }$ allows us to reformulate the policy evaluation step as a convex leastsquares problem with a closed-form solution by optimizing the parameters using (8) with $p \ = \ 2$ . Importantly, this requires that we fix the policy at the next timestep. In the simulated case study, we use the same Q-function parametrization for LSPI as in the proposed method, i.e., (10). Note that LSPI cannot include a positive definite constraint on S.
