# 6.3.4 Collision Checking in $\bar { \mathcal { A } } _ { \sf f r e e } ( \bar { e } _ { t r } , \bar { \eta } _ { r } )$

Collision checking for points $q \in \mathbb { T }$ in variants of the standard RRT algorithm is performed by checking whether q belongs to $\mathcal { A } _ { \mathrm { f r e e } }$ or not.
For a line segment $X _ { \mathrm { L i n e } }$ connecting two nodes of $\nu ,$ the latter is usually discretized into a finite set of points, which are checked separately for collision (e.g., [320, 369]).
Another approach is to consider an over-approximation of the convex hull of the points that form $X _ { \mathrm { L i n e } } ~ ( [ 3 7 0 ] )$ .
In our case, however, we are interested in checking for collisions in the extended free space $\bar { \lambda } _ { \mathrm { f r e e } } ( \bar { e } _ { t r } , \bar { \eta } _ { r } )$ .
Recall that the proposed feedback control scheme guarantees that $q ( t ) \in \mathcal { P } ( q _ { \mathrm { d } } ( t ) , ( \bar { e } _ { t r } , \bar { \eta } _ { r } ) )$ ) for any trajectory $q _ { \mathrm { d } } ( t )$ , formed by the several line segments $X _ { \mathrm { L i n e } }$ that connect the nodes in $\nu$ sampled in Algorithm 2.
Therefore, checking whether the points $q _ { s } \in X _ { \mathrm { L i n e } }$ belong to $\mathcal { A } _ { \mathrm { f r e e } }$ is not sufficient.
That is, for each such point $q _ { s } \in X _ { \mathrm { L i n e } }$ , one must check whether $z \in \mathcal { A } _ { \mathrm { f r e e } }$ , $\forall z \in \mathcal { P } ( q _ { s } , ( \bar { e } _ { t r } , \bar { \eta } _ { r } ) )$ , which is equivalent to checking if $q _ { s } \in \bar { \mathcal { A } } _ { \mathrm { f r e e } } ( \bar { e } _ { t r } , \bar { \eta } _ { r } )$ .
There are two procedures that one can use for that.
Firstly, for each $q _ { s } ,$ , a finite number of points z can be sampled from a uniform distribution in $\mathcal { P } ( q _ { s } , ( \bar { e } _ { t r } , \bar { \eta } _ { r } ) )$ and separately checked for collision.
Then, for a sufficiently high number of such samples, and assuming a certain “fat”-structure of the workspace obstacles (e.g., there are no long and skinny obstacles such as wires, cables and tree branches, etc., see ([371]) for more details), this approach can be considered to be complete, i.e., the resulting path will belong to the extended free space $\mathcal { A } _ { \mathrm { f r e e } }$ .
Secondly, we calculate the limit poses of each link of the robot, based on the lower and upper bounds by the joints that affect it, as defined by $( \bar { e } _ { t r } , \bar { \eta } _ { r } )$ .

Subsequently, we compute the convex hull of these limit poses, which is expanded by an appropriate constant to yield an over-approximation of the swept volume of the potential motion of the link, as described in [370].
The resulting shape is then checked for collisions for each link separately.
