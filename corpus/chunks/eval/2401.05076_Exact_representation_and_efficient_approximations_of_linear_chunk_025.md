# 4.2. Robustness of HTNN and U-HTNN architectures

Following [4, 15], HTNN and U-HTNN have Lipschitz behavior with Lipschitz constant $\begin{array} { r } { \mathbf { L } = \prod _ { k = 1 } ^ { \zeta } \left\| W ^ { [ k ] } \right\| } \end{array}$ . Thus, given an input x0 and some perturbation ε, we can majorize the perturbation on the output via the inequality:

$$\left| \left| \mathbf {u} ^ {\mathrm{NN}} (x _ {0} + \boldsymbol {\varepsilon}, \hat {\boldsymbol {\Theta}}) - \mathbf {u} ^ {\mathrm{NN}} (x _ {0}, \hat {\boldsymbol {\Theta}}) \right| \right| \leq \mathbf {L} \| \boldsymbol {\varepsilon} \|,$$

where ${ \mathbf u } ^ { \mathrm { N N } } \in \{ { \mathbf u } ^ { \mathrm { H T N N } } , { \mathbf u } ^ { \mathrm { U - H T N N } } \}$ . Therefore, Lipschitz constant L can be used as a certificate of robustness of our networks provided that it is tightly estimated. Tighter bounds for Lipschitz constant exist but at the price of more complex computations, see e.g., [7].
