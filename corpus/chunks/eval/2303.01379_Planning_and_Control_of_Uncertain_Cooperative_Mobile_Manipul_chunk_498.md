• Sample(i): Samples $q _ { \mathrm { r a n d } }$ from a uniform distribution in the extended free space $\bar { \lambda } _ { \mathrm { f r e e } } \big ( \big ( \bar { e } _ { t r } , \bar { \eta } _ { r } \big ) \big )$ , where $\bar { e } _ { t r } , \bar { \eta } _ { r }$ are the constants from (6.9) that define the funnel polyhedron $\mathcal { P } ( q _ { \mathrm { d } } ( t ) , ( \bar { e } _ { t r } , \bar { \eta } _ { r } ) )$ (see (6.10)) around a reference trajectory $q _ { \mathrm { d } } ( t )$ that $q ( t )$ can evolve in.

• Nearest $( { \mathcal { G } } , q )$ : Finds the node $q _ { \mathrm { n e a r e s t } }$ in the tree such that $d _ { \mathbb { T } } ( q _ { \mathrm { n e a r e s t } } , q )$ $= \mathrm { m i n } _ { z \in \mathcal { V } } d _ { \mathbb { T } } ( z , q )$ .

• Steer $( q , z )$ : Computes a point $q _ { \mathrm { n e w } }$ lying on the straight line from z to q such that $d _ { \mathbb { T } } ( q , q _ { \mathrm { n e w } } ) = \epsilon$ , where  is a tuning constant that represents the incremental distance from q to $q _ { \mathrm { n e w } }$ .

• ObstacleFree $( q , z )$ : Checks whether the path $X _ { \mathrm { L i n e } } : [ 0 , \sigma ]  \mathbb { T }$ , for some positive σ, from q to z is collision free with respect to the extended free space, i.e., check whether $q ^ { \prime } \in \bar { \mathcal { A } } _ { \mathrm { f r e e } } \big ( ( \bar { e } _ { t r } , \bar { \eta } _ { r } ) \big ) , \forall q ^ { \prime } \in X _ { \mathrm { L i n e } }$ .

The difference hence of B-RRT with respect to the standard RRT algorithm is the use of the extended free space $\bar { \lambda } _ { \mathrm { f r e e } } \big ( \big ( \bar { e } _ { t r } , \bar { \eta } _ { r } \big ) \big )$ in the procedures of sampling new points (function Sample) and checking collisions of the path between two nodes (function ObstacleFree).
