In order to demonstrate the effect of the choice of the mesh, another parameter identification experiment was conducted using the same training and query datasets but using a shifted mesh that does not contain the true parameter values $\delta = 0 , \alpha = 1$ , and $\beta = - 1$ . In this case we use a mesh composed of equally spaced nodes in $[ - 3 . 3 , 2 . 7 ] \times$ $[ - 3 . 3 , 2 . 7 ] \times [ - 3 . 3 , 2 . 7 ] ] \subset \mathbb { R } ^ { 3 }$ with a spacing of 0.5. Figure 4 shows values of the MSE against the Euclidean distance from the true parameter in the parameter space, similar to 3. In this case, it can be seen that the parameter values that produce the smallest MSE $( { \hat { \alpha } } = 0 . 7 , { \hat { \beta } } = - 1 . 3 ,$ and $\hat { \delta } = - 0 . 3 . )$ is not the closest node in the mesh to the true values in Euclidean distance, but we still see a separation between a cluster of ‘low-error’ parameter vectors and other parameter vectors on the mesh.
