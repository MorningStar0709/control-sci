(b) Approach conic corridor : $\| \tilde { e } _ { p } \| c o s ( \alpha _ { c o n e } ) \leq \hat { 1 } _ { t } \tilde { e } _ { p }$   
Confines the chaser within a cone (half-angle $\alpha _ { c o n e } )$ aligned with the target’s docking axis $\hat { 1 } _ { t }$ during terminal approach as illustrated in Fig. 1(a).   
(c) Field of view constraint: $\lVert \tilde { e } _ { F O V } \rVert c o s ( \alpha _ { F O V } ) \le \hat { \boldsymbol { 1 } } _ { c } \tilde { e } _ { F O V }$ Ensures continuous target visibility within the chaser’s sensor cone half-angle $\alpha _ { F O V }$ along its docking axis $\hat { 1 } _ { c }$ as depicted in Fig. 1(b), where $\tilde { e } _ { F O V }$ is the relative position vector of target satellite w.r.t chaser satellite.   
(e) Velocity Constraint: This constraint enforce a constant velocity of $0 . 3 m / s$ during mid-range proximity maneuver from 1 km to 10 m and reduces it to $0 . 0 3 m / s$ for final soft docking.

$$
\tilde {e} _ {v} = \left\{ \begin{array}{l l} 0. 3 m / s & , i f 1 0 m <   \| \tilde {e} _ {p} \| \leq 1 0 0 0 m \\ 0. 0 3 m / s & , i f \| \tilde {e} _ {p} \| \leq 1 0 m \end{array} \right.
$$

(e) Impulse Constraint: Since the maneuver is impulsive in nature, the focus is on minimizing the change in velocity $( \triangle v )$ rather than the applied force, as the force is not explicitly part of the state vector. The control inputs are modeled as instantaneous impulses, making $\triangle v$ the primary metric of interest for optimization.

$$- 3 0 0 \sqrt {3} < \| \triangle v \| < 3 0 0 \sqrt {3}$$
