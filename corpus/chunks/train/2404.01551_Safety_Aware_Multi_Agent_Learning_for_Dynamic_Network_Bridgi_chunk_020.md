Base connectivity reward: The base reward, $R _ { \mathrm { b a s e } }$ , encourages larger connected components:

$$R _ {\text { base }} (s) = \frac {| C _ {\max} (s) |}{| \mathcal {V} |}, \tag {17}$$

where $\left| C _ { \operatorname* { m a x } } ( s ) \right|$ represents the size of the largest connected component in state $s ,$ and |V | is the total number of entities.

Centroid distance penalty: The centroid distance penalty, $P _ { \mathrm { c e n t } }$ , is based on agents’ positions relative to the targets, penalizing them based on the Euclidean distance between the centroid of the agents’ positions and the central point between the targets.

Target path bonus: A bonus, $B _ { \mathrm { p a t h } } = 1 0 0$ , is awarded if a path exists between the two targets.

The overall reward combines these three elements:

$$
R (s, a) = \left\{ \begin{array}{l l} B _ {\text { path }} (s) & \text { if } \exists \text { path } (T _ {1}, T _ {2}); \\ R _ {\text { base }} (s) - P _ {\text { cent }} (s), & \text { otherwise }. \end{array} \right. \tag {18}
$$

Simulated UAV dynamics: For our experiments, we use a six-degrees-of-freedom (6-DOF) unmanned aerial vehicle (UAV) as an agent, where its dynamics, control laws, and safety sets used in our simulations are defined in the Appendix of [7]. For the UAV simulations, the nonlinear model of the drone is used.

Target mobility: To increase task complexity, each target moves deterministically toward waypoints located outside the communication range of any agent. When a target reaches a waypoint, it pauses for a fixed number of time-steps (5 in our implementation) before selecting a new one. Waypoint selection ensures that at least two agents (and up to $N )$ are required to maintain a communication bridge between the targets.
