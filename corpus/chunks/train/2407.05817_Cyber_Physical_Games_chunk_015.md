Assuming both agents issue messages at all points in time within their frames of reference and that their frames of reference are synchronized with each other, the environment processes messages issued by both agents in the same order as issued, but with random ordering within message pairs. I.e., give two messages m0 and $m _ { 1 }$ emitted by agents $c _ { 0 }$ and $c _ { 1 }$ synchronously, the environment may process first $m _ { 0 } .$ , first $m _ { 1 } ,$ or both at the same time (the latter is equivalent to a traditional turn-based game, where both players must commit to an action at any turn, and both actions are executed concurrently). Because the game is infinite, we construct not a game tree from the current to terminal states, but rather a closed game graph, where payoffs are associated with specific walks within the graph.
