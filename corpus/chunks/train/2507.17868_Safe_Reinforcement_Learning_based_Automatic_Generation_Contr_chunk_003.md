# II. BACKGROUND

CBFs seek to establish a safety set, which represents the states in which the system is considered safe, and enforce the forward invariance of this safety set, meaning that the system is forced to remain within this set for all future times.

In control applications, CBF permits only control actions that maintain the system within the defined safety set. When integrated into the learning process of RL agents, CBF can ensure that the agent’s actions adhere to safety constraints, thereby promoting a safe learning process. The safe set is defined by the super-level set of a continuously differentiable function $h : \mathbb { R } ^ { n }  \mathbb { R }$ :

$$\mathcal {S} = \{x \in \mathbb {R} ^ {n}: h (x) \geq 0 \}\partial \mathcal {S} = \{x \in \mathbb {R} ^ {n}: h (x) = 0 \} \tag {1}\operatorname{Int} (\mathcal {S}) = \{x \in \mathbb {R} ^ {n}: h (x) > 0 \}$$

In this paper, we employ logarithmic barrier functions, inspired by the work in [21], [22]. The function B(x) is considered a logarithmic barrier function if there exists a strictly increasing function $\alpha : \mathbb { R } $ R with $\alpha ( 0 ) = 0$ , such that for the control system ${ \dot { x } } = f ( x ) + g ( x ) u$ , the function $B ( x )$ satisfies the inequality:

$$\dot {\mathcal {B}} = \frac {\partial \mathcal {B}}{\partial x} \dot {x} = \frac {\partial \mathcal {B}}{\partial x} (f (x) + g (x) u) \leq \alpha \left(\frac {1}{\mathcal {B} (x)}\right) \tag {2}$$

The logarithmic barrier function $B ( x )$ is computed based on h as follows:

$$\mathcal {B} (x) = - \log \left(\frac {h (\boldsymbol {x})}{1 + h (\boldsymbol {x})}\right) \tag {3}$$

and exhibits two characteristics:

$$\inf _ {x \in \mathcal {S}} \mathcal {B} (x) > 0 \tag {4}\lim _ {x \to \partial \mathcal {S}} \mathcal {B} (x) = \infty \tag {5}$$
