# Remark 5.5 (Steps on determination of non-overshooting controller )

For the system (39), the flow chart of controller design is explained in Fig. 4. Furthermore, the the steps on controller determination are described as follows.

Step 1: Measure the system initial states, and get the initial errors $e _ { 1 } \left( 0 \right)$ and $e _ { 2 } \left( 0 \right)$ .

Step 2: Select $e _ { 1 c } \in ( 0 , k _ { 2 M } - L _ { d } )$ and $e _ { 2 c } \in \left( e _ { 1 c } , \sqrt { \left( k _ { 2 M } - L _ { d } \right) e _ { 1 c } } \right]$ , respectively, and

$$k _ {c} > L _ {d}, \rho_ {c} = \rho_ {c 0} \frac {1}{2} \ln \frac {k _ {c} + L _ {d}}{k _ {c} - L _ {d}} \tag {55}$$

where, $k _ { 2 M } > L _ { d }$ is the up-bound of $k _ { 2 }$ from the system gain limitation; $\rho _ { c 0 } > 1$ .

Step 3: Determine the controller parameters $\boldsymbol { k } _ { 1 } , \boldsymbol { k } _ { 2 }$ and $\rho$ through the calculations in Equations $( 3 4 ) { \sim } ( 3 8 )$ . [Note: When the known reference jumps or suddenly changes, the parameters $\boldsymbol { k } _ { 1 } , \ \boldsymbol { k } _ { 2 }$ and $\rho$ are updated]

Step 4: Controller output:

$$
u (t) = \left\{ \begin{array}{l} k _ {c} \tanh \left[ \rho_ {c} \left(e _ {2} (t) + e _ {2 c} \operatorname{sign} \left(e _ {1} (t)\right)\right) \right] - h (t), \text {if} | e _ {1} (t) | > e _ {1 c}; \\ k _ {2} \tanh \left[ \rho \left(e _ {2} (t) + k _ {1} e _ {1} (t)\right) \right] - h (t), \text {if} | e _ {1} (t) | \leq e _ {1 c} \end{array} \right. \tag {56}
$$
