$$\omega_ {k} ^ {*} = \frac {u _ {k} ^ {*}}{v \cos \varepsilon_ {H , k}}, \tag {56}$$

where ?? is the vehicle’s linear velocity and $\varepsilon _ { H , k }$ is the heading error at time ??. Note that the matrices $\left( \mathbf { M } ^ { \top } \mathbf { Q } \mathbf { M } + \mathbf { R } \right) ^ { - 1 }$ and $\mathbf { M } ^ { \top } \mathbf { Q }$ are constants at all time steps and thus can be precomputed to significantly reduce real-time computational complexity, which is a key difference between the FBLMPC strategy and traditional nonlinear MPC.

5.2.2 GP-FBLMPC. In GP-FBLMPC, the focus shifts to using feedback linearized (FBL) states $\mathbf { z } _ { k }$ as the system variables, in contrast to the robot pose states $\mathbf { x } _ { k }$ used in GP-NMPC. Utilizing (41), the relationship between the actual and predicted robot pose state $\mathbf { x } _ { k + 1 }$ is expressed as:

$$\mathbf {x} _ {k + 1} - \mathbf {f} \left(\mathbf {x} _ {k}, \mathbf {u} _ {k}\right) = \mathbf {g} \left(\mathbf {x} _ {k}, \mathbf {v} _ {k - 1}, \mathbf {u} _ {k}, \mathbf {u} _ {k - 1}\right). \tag {57}$$

where the right side of (57) are the FBL states $\mathbf { z } _ { k + 1 }$ . The GP model incorporating these FBL states is formulated as:

$$\mathbf {z} _ {k + 1} = \mathbf {g} (a _ {k}), \tag {58}$$
