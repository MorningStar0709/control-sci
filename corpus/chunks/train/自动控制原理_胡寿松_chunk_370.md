本例设计结果为 $z = 3.5, p = 25, K_{1} = 1800$ ，于是

$$G _ {c} (s) = \frac {1 8 0 0 (s + 3 . 5)}{s + 2 5}$$

校正后系统的阶跃响应表明,系统引入超前校正网络后,相角裕度明显增大,动态性能改善较大,超调量及调节时间已能满足设计要求,但校正后系统的静态速度误差系数

$$K _ {v} = \frac {1 8 0 0 \times 3 . 5}{5 \times 1 0 \times 2 5} = 5. 0 4 < 1 0$$

使得系统斜坡响应的稳态误差高达 20%，不符设计指标要求。

为了减少系统的稳态误差,可尝试用根轨迹法设计滞后校正网络,其传递函数为

$$G _ {c} (s) = \frac {K _ {1} (s + z)}{s + p}, \quad (| p | < | z |)$$

设计滞后校正网络的已知条件为 $\zeta=0.59$ 和 $\omega_{n}=2.49$ ，由此可确定闭环主导极点的区域。滞后校正网络的设计步骤可归纳如下：

1) 绘制未校正系统的根轨迹。

2) 根据 $\zeta=0.59, \omega_{n}=2.49$ 确定预期主导极点的容许区域，并进一步在未校正系统的根轨迹上确定校正后的预期主导极点。

以上两步的程序文本如图 6-41 所示。

```txt
G0=tf([1],[1,15,50,0]);
rlocus(G0);hold on;

xc=[-10:0.01:-zeta*wn];c=sqrt(wn^2-xc.^2); 
```

4）计算 $b = K_{v} / K_{v}^{\prime}$ ，本例要求 $K_{v}\geqslant 10$   
5）根据求得的 $b$ ，配置滞后校正网络的零、极点，使校正后系统的根轨迹经过预期主导极点。

以上各步见图6-42所示程序文本。在本例设计过程中，根据选定的预期主导极点，采用rlocfind函数计算根轨迹增益的取值，得到 $K_{1} = 100$ ；为了满足 $K_{v} \geqslant 10$ 的设计要求，计算合适的 $b$ 取值，得到 $b = 10$ ；在配置滞后校正网络的零、极点时，取零点 $z = -0.1$ ，极点 $p = -0.01$ ，以免明显改变校正后系统的根轨迹形状。于是，所设计的滞后校正网络为

$$G _ {c} (s) = \frac {1 0 0 (s + 0 . 1)}{s + 0 . 0 1}$$
