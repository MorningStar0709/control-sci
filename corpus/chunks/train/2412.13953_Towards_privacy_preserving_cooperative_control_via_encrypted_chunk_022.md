Since multiple keys are involved in the encrypted ADMM implementation, we briefly summarize the key management. Importantly, only the operator holds sk0. However, since computations are carried out by the agents and since communications are secured by AES, the operator gets no access to corresponding ciphertexts. Now, each agent i holds the secret key ski to its own instance of the cryptosystem, which is required for the final decryption of $[ [ \alpha _ { i } ^ { \ell } ] ] _ { i }$ . In order to perform the preceding key switch, for each agent i, one selected neighbor obtains the switching key in terms of $\mathsf { E n c } _ { i } ( \mathsf { s k } _ { 0 } )$ .
