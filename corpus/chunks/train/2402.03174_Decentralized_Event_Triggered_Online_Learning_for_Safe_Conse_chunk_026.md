# B. Performance among Difference Controllers

First, we show the performance for each cases with $\pmb { x } ( 0 ) =$ $[ - 0 . 5 2 , 0 . 1 5 , - 0 . 0 6 , - 0 . 7 1 ] ^ { T }$ resulting in $\bar { x } ^ { * } = - 0 . 2 8 5$ .

The state evolution for each agent over time is shown in Fig. 1. The performance by using the conventional controller (2) is shown in subfigures (a) and (b), where the states almost achieve consensus with some errors but do not converge to the average states ${ \bar { x } } ^ { * }$ . The bias between the actual converge state $\begin{array} { r } { \bar { x } _ { \infty } = \operatorname* { l i m } _ { t \to \infty } \mathbf { 1 } _ { N } ^ { T } \pmb { x } ( t ) } \end{array}$ and the average states ${ \bar { x } } ^ { * }$ c a n hardly be quantified. Therefore, it is not suitable for safety critical scenarios, since the actual converge state $\bar { x } _ { \infty }$ may not in the pre-defined domain X. Moreover, it is intuitive that with online learning the actual converge state is closer to the average states, because of the smaller prediction error compared to the offline learning. With the proposed control using (3) and (4), the bias between $\bar { x } _ { \infty }$ and ${ \bar { x } } ^ { * }$ is guaranteed to be reduced and shown in subfigures (c) and (d) in Fig. 1, guarantee the effectiveness of Theorem 1 as in Remark 6. While the states $x _ { i }$ tend to close ${ \bar { x } } ^ { * }$ in (c), the bias to ${ \bar { x } } ^ { * }$ , i.e., $| \bar { x } _ { \infty } - \bar { x } ^ { * } |$ , is obvious due to the poor prediction under offline learning. Additionally, with online learning as in (d), the controlled system achieves a smaller average consensus error compared to case (c).
