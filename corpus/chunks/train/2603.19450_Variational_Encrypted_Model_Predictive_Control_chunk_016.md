Since both $L _ { U }$ and Γ are independent of the state $x _ { 0 } ,$ , the client encrypts and transmits $\mathsf { E n c } ( L _ { U } )$ and Enc(Γ) to the cloud during the offline phase. The cloud then generates Gaussian noise samples $\xi ^ { ( i ) } \sim \mathcal { N } ( 0 , I )$ , for $i = 1 , \ldots , K$ , and precomputes the following encrypted quantities2

$$\mathbf {c} _ {L U} ^ {(i)} := \operatorname{Enc} (L _ {U}) \otimes \operatorname{Enc} (\xi^ {(i)}), \tag {25}\mathbf {c} _ {\Gamma} ^ {(i)} := \operatorname{Enc} (\Gamma) \otimes \operatorname{Enc} (\xi^ {(i)}).$$

Since these quantities depend only on the system model and not on the current state, this preprocessing is performed once. Moreover, these precomputed values can be cached for use during the online phase.

2) Online execution: At each MPC update step t, with $x _ { 0 } : = x ( t )$ , the client encrypts and transmits Enc $( m _ { U } ( x _ { 0 } ) )$ and $\mathsf { E n c } ( b ( x _ { 0 } ) )$ to the cloud. Then, for each $i = 1 , \ldots , K$ , the cloud retrieves the cached quantities $\mathbf { c } _ { L U } ^ { ( i ) }$ and $\mathbf { c } _ { \Gamma } ^ { \left( i \right) }$ to form the encrypted trajectory samples

$$\mathbf {U} ^ {(i)} = \mathsf {E n c} (m _ {U} (x _ {0})) \oplus \mathbf {c} _ {L U} ^ {(i)}, \tag {26}$$

together with the encrypted constraint residuals

$$\mathbf {g} (U ^ {(i)}; x _ {0}) = \mathsf {E n c} (b (x _ {0})) \oplus \mathbf {c} _ {\Gamma} ^ {(i)}. \tag {27}$$

To evaluate $s _ { \ell } ( U ^ { ( i ) } ; x _ { 0 } )$ homomorphically, the cloud first applies the polynomial surrogate $h _ { \ell }$ to $\mathbf { g } ( U ^ { ( i ) } ; x _ { 0 } )$ , followed by a standard sum-reduction procedure [18, Section 4.1] that iteratively rotates and adds the ciphertext to compute the aggregated score, which we denote by $\mathbf { s } _ { \ell } ( U ^ { ( i ) } ; x _ { 0 } )$ .

The cloud returns $\{ \mathbf { U } ^ { ( i ) } , \mathbf { s } _ { \ell } ( U ^ { ( i ) } ; x _ { 0 } ) \} _ { i = 1 } ^ { K }$ 1 back to the client. After decryption, the client computes the thresholded scores and corresponding desirability weights to construct the estimator $\tilde { U } _ { K , \ell } ( \boldsymbol { x } _ { 0 } )$ , as in Lines 9–14 of Algorithm 2. Finally, the first control input is applied to the system.
