# IV. DISTRIBUTED EVENT-TRIGGERED CONSENSUS UNDER LQ PERFORMANCE CONSTRAINTS

In this section, we propose a distributed event-triggered consensus control method under the LQ performance constraint.

First, we introduce several notations used throughout this paper. Let $\begin{array} { r } { \begin{array} { l c l } { e _ { i } [ k ] } & { : = } & { \hat { x } _ { i } [ k ] \ - \ x _ { i } [ k ] } \end{array} } \end{array}$ and $e [ k ] \ : = { }$ $[ e _ { 1 } ^ { \top } [ k ] ~ . . . ~ e _ { N } ^ { \top } [ k ] ] ^ { \top }$ . We consider the following variable:

$$\bar {x} _ {i} [ k ] = A \hat {x} _ {i} [ k - 1 ] + B \hat {u} _ {i} [ k - 1 ], \tag {17}$$

which serves as the a priori prediction of $x _ { i } [ k ]$ , with the initial state $\bar { x } _ { i } [ 0 ] = 0$ . Accordingly, ${ \hat { x } } _ { i } [ k ]$ is updated by

$$
\hat {x} _ {i} [ k ] = \left\{ \begin{array}{l l} x _ {i} [ k ] & \text { if } k = t _ {\ell} ^ {i} \text { for } \ell \in \mathbb {Z} _ {\geq 0}, \\ \bar {x} _ {i} [ k ] & \text { otherwise }. \end{array} \right.
$$

We also define $\bar { e } _ { i } [ k ] : = \bar { x } _ { i } [ k ] - x _ { i } [ k ]$ and $\bar { e } [ k ] : =$ $[ \bar { e } _ { 1 } ^ { \top } [ k ] \enspace \dots \enspace \bar { e } _ { N } ^ { \top } [ k ] ] ^ { \top }$ . In addition, let us consider

$$
\begin{array}{l} \hat {\phi} _ {i} [ k ] := \frac {1}{2} \sum_ {j \in \mathcal {N} _ {i}} a _ {i j} (\hat {x} _ {i} [ k ] - \hat {x} _ {j} [ k ]) ^ {\top} Q (\hat {x} _ {i} [ k ] - \hat {x} _ {j} [ k ]) \\ + c ^ {2} \hat {\zeta} _ {i} ^ {\top} [ k ] F ^ {\top} R F \hat {\zeta} _ {i} [ k ], \tag {18} \\ \end{array}
$$

which depends only on locally available information to agent i.

Suppose that each agent shares its state at time $k = 0 ,$ , $\mathrm { i } . \mathrm { e } . , t _ { 0 } ^ { i } = 0$ for all $i \in \{ 1 , \ldots , N \}$ . Then, we have $\hat { x } _ { i } [ 0 ] =$ $x _ { i } [ 0 ]$ . Using (18), each agent asynchronously determines transmission instants $\{ t _ { \ell + 1 } ^ { i } \} _ { \ell \in \mathbb { Z } _ { \geq 0 } }$ as follows:

$$t _ {\ell + 1} ^ {i} = \min \{k > t _ {\ell} ^ {i} \colon \bar {e} _ {i} ^ {\top} [ k ] \Omega_ {i} \bar {e} _ {i} [ k ] > \sigma \hat {\phi} _ {i} [ k - 1 ] \}, \tag {19}$$

where $\Omega _ { i } \succ 0$ and $\sigma > 0$ are design parameters. Intuitively, agent i triggers when the weighted prediction error becomes too large compared to the local estimated stage cost. This is because a larger disagreement among agents can tolerate a larger prediction error, whereas near consensus even a small prediction error can have a relatively large impact on the LQ performance.
