# A. Settings of Components

This subsection briefly states the settings of each of the components constituting the NCS Σ in Fig. 1.

Plant $P _ { \mathrm { c } } \mathrm { : }$ : We suppose $P _ { \mathrm { c } }$ is a continuous-time deterministic linear plant represented by the state equation

$$\dot {x} _ {\mathrm{c}} (t) = A _ {\mathrm{c}} x _ {\mathrm{c}} (t) + B _ {\mathrm{c}} u _ {\mathrm{c}} (t), \tag {1}$$

where $t \in \mathbf { R }$ denotes the continuous time with initial time $t = 0$ , and $x _ { \mathrm { c } } ( t ) \in \mathbf { R } ^ { n }$ and $u _ { \mathrm { c } } ( t ) \in \mathbf { R } ^ { m }$ are the state and the input, respectively. The initial state $x _ { \mathrm { c } } ( 0 ) = x _ { 0 }$ is assumed to be given.

Sampler S and Hold H: We suppose S and H are the sampler and the zero-order hold that operate in synchronization at sampling time instants $t _ { k } ~ \left( k \in \mathbf { N } _ { 0 } \right)$ satisfying

$$t _ {0} = 0, \quad t _ {k + 1} - t _ {k} > 0, \quad \lim _ {k \rightarrow \infty} t _ {k} = \infty . \tag {2}$$

We use the symbol t for representing continuous time, and k for discrete time. The sampling interval $h _ { k }$ is defined as

$$h _ {k} := t _ {k + 1} - t _ {k}. \tag {3}$$

As is obvious from this definition, the sampling interval of our NCS is not constant but time-varying. The relation of the sampling interval with communication delays will be described later.

Controller Ψ: In this paper, we deal with a discrete-time state-feedback controller for the NCS Σ. For simplicity, we suppose that the controller Ψ determines the control input $u _ { k }$ immediately after receiving the sampled output $x _ { k } = x _ { \mathrm { c } } ( t _ { k } )$ of the plant at each discrete time k. The class of the controller and its design method will be discussed later.
