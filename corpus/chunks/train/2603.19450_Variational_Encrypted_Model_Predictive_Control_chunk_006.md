$$
\begin{array}{l} \left\| \operatorname{Dec} (\operatorname{Enc} (\mathrm{pt})) - \mathrm{pt} \right\| \leq B ^ {\text { Enc }}, \\ \left\| \operatorname{Dec} \left(\mathbf {c} \oplus \mathbf {c} ^ {\prime}\right) - \left(\operatorname{Dec} (\mathbf {c}) + \operatorname{Dec} \left(\mathbf {c} ^ {\prime}\right)\right) \right\| = 0, \\ \left\| \operatorname{Dec} \left(\operatorname{Enc} \left(\mathrm{pt}\right) \otimes \operatorname{Enc} \left(\mathrm{pt} ^ {\prime}\right)\right) - \mathrm{pt} \odot \mathrm{pt} ^ {\prime} \right\| \\ \leq \left(\| p t \| + \| p t ^ {\prime} \|\right) B ^ {E n c} + \left(B ^ {E n c}\right) ^ {2} + B ^ {M u l t} =: B ^ {\otimes} \left(\| p t \|, \| p t ^ {\prime} \|\right), \\ \left\| \operatorname{Dec} \left(\operatorname{RotCt} _ {\rho} (\mathbf {c})\right) - \operatorname{RotVec} _ {\rho} (\operatorname{Dec} (\mathbf {c})) \right\| \leq B ^ {\text { Rot }}. \\ \end{array}
$$

By sequentially composing the operations from Lemma 1, one can homomorphically evaluate an arbitrary polynomial [14], [15]. With a slight abuse of notation, let $h _ { \ell } ( { \mathbf { c } } ) \in { \mathcal { C } }$ denote the homomorphic evaluation of a polynomial $h _ { \ell }$ on a ciphertext $\mathbf { c } \in { \mathcal { C } }$ . The following corollary provides a bound on the resulting error growth.

Corollary 1: Consider a polynomial1 $\begin{array} { r l r l } { h _ { \ell } ( \gamma ) } & { { } } & { = } & { } \end{array}$ $\scriptstyle \sum _ { k = 0 } ^ { \ell } c _ { k } \gamma ^ { k }$ of order $\ell \in \mathbb { N }$ , where $c _ { k } \in \mathbb { C }$ for $k = 0 , \ldots , \ell .$ For any $\mathbf { c } \in { \mathcal { C } } .$ ,

$$
\begin{array}{l} \left\| \operatorname{Dec} (h _ {\ell} (\mathbf {c})) - h _ {\ell} (\operatorname{Dec} (\mathbf {c})) \right\| \\ \leq \ell \cdot B ^ {\text { Mult }} \max (1, \| D e c (\mathbf {c}) \|) ^ {\ell - 1} \sum_ {k = 0} ^ {\ell} | c _ {k} | =: B ^ {\text { poly }} (h _ {\ell}, \mathbf {c}). \\ \end{array}
$$

The parameters $N _ { \mathrm { c t } }$ and $\Delta$ affect not only the error growths in Lemma 1 and Corollary 1 but also the security level of the scheme. Typically, increasing $N _ { \mathrm { c t } }$ or decreasing $\Delta$ strengthens security, potentially at the cost of higher computational cost. In this letter, we assume that these parameters are given based on standard security requirements, and focus on integrating the CKKS scheme into our control framework. For detailed parameter selection methods, see [14].
