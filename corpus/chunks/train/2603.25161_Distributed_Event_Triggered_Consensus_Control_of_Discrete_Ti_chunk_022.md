$$\varepsilon^ {\star} \in \arg \max _ {\varepsilon \in \mathcal {E}} \sigma^ {\star} (\varepsilon),$$

and set $\boldsymbol { \sigma } = \boldsymbol { \sigma } ^ { \star } ( \boldsymbol { \varepsilon } ^ { \star } )$ and

$$\Omega_ {i} = \Omega_ {i} ^ {\star} (\varepsilon^ {\star}), \quad \eta = \eta^ {\star} (\sigma^ {\star}; \varepsilon^ {\star}), \quad \delta = \delta^ {\star} (\sigma^ {\star}; \varepsilon^ {\star})$$

according to (44), (45), and (48), respectively.

Remark 5: The proposed parameter design method is a heuristic procedure, and hence the obtained parameters do not necessarily yield the largest possible value of σ. Likewise, while we aim to reduce the number of transmissions by enlarging σ, it does not in itself guarantee the smallest number of transmissions. Instead, the design method in this section provides a tractable way to search for parameters that satisfy the given performance constraint and may lead to reduced communication in practice.
