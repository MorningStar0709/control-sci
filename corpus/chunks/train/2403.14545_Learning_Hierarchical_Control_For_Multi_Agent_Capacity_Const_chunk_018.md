# V. HIERARCHICAL LMPC

We now introduce our proposed data-driven policy formulations and parameter update procedures. Each level of the hierarchical controller utilizes a data-driven MPC policy. We adapt ideas from Learning Model Predictive Control (LMPC), an approach for iteratively improving on an initial provided feasible trajectory [26]. The technique relies on the realization that any state in a given feasible trajectory is a subset of the maximum controllable set to the task goal. Closed-loop data collected at each iteration is used to successively improve the construction of an MPC terminal set and terminal cost, resulting in iterative closed-loop performance improvement while maintaining feasibility guarantees. Here we demonstrate how to adapt this idea for safe data-driven hierarchical control.

As defined in (19), we denote with $\mathbf { x } ^ { r }$ the agent closedloop trajectory during iteration $r ,$ and with $\mathbf { x } ^ { H , r } = g _ { \tau } ( \mathbf { x } ^ { r } )$ the corresponding sequence of high-level states. At the beginning of each iteration r, the trajectories collected in the previous iterations are used to formulate data-driven policies $\cdot ^ { - } \pi ^ { - r }$ and $\pi ^ { L , r }$ and their parameters $S ^ { L , r } , S ^ { H , r } , \hat { \theta } ^ { r }$ , and $\hat { \Theta } ^ { r }$ .
