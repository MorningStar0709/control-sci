Now, the encrypted ADMM scheme requires one crucial additional element, which is not necessary in the plaintext version. In fact, as indicated above, agent i shall get access to $\alpha _ { i } ^ { \ell }$ after evaluating a user-defined number of ℓ ADMM iterations (via $\mathrm { A l g } . 3 )$ . Hence, a final key switch $[ [ \alpha _ { i } ^ { \ell } ] ] _ { 0 \to i }$ is required once. Clearly, this switch must not be performed by agent i since it requires $\mathsf { E n c } _ { i } ( \mathsf { s k } _ { 0 } )$ and since agent i holds $\mathsf { s k } _ { i } .$ . Thus, given $\mathsf { E n c } _ { i } ( \mathsf { s k } _ { 0 } )$ , agent i could apply $\mathsf { D e c } _ { i }$ to obtain sk0, which would render the encrypted implementation of the algorithms useless. As a remedy, agent i selects a neighbor $j \in \mathcal N _ { i }$ and sends $[ [ \alpha _ { i } ^ { \ell } ] ] _ { 0 }$ to it secured by AES. The neighbor then performs the key switch $[ [ \alpha _ { i } ^ { \ell } ] ] _ { 0 \to i }$ and returns $[ [ \alpha _ { i } ^ { \ell } ] ] _ { i }$ (without the need of an additional encryption) to agent i, who is finally able to decrypt (and apply) the result. Remarkably, in the preceding iteration $\tau = \ell - 1$ , Algorithm 3 can be terminated after the first step since the results of all upcoming steps are of no avail.
