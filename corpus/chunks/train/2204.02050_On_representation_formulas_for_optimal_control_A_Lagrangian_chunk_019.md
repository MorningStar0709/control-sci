In general, obtaining $\tilde { u } [ k ]$ satisfying the conditions in (ii) is not an easy task. As a practical remedy, we propose an intuitive method that uses a δ-net to approximate $\beta [ k ] \in \operatorname { C o n v } ( B ( x [ k ] , t _ { k } ) )$ obtained from the optimization problem by $- f ( t _ { k } , x [ k ] , \bar { u } [ k ] )$ , where ${ \bar { u } } [ k ] \in U _ { \delta }$ . Specifically, having found $\beta [ k ]$ and $x [ k ]$ , we choose $\bar { u } [ k ] \in U _ { \delta }$ so that $- f ( t _ { k } , x [ k ] , \bar { u } [ k ] )$ becomes the closest point to $\beta [ k ]$ . Using this idea, Step (ii) can be modified as

(ii0) Approximate $\tilde { u } [ k ]$ as

$$\tilde {u} [ k ] \approx \bar {u} [ k ] \in \underset {a \in U _ {\delta}} {\arg \min} | \beta [ k ] + f (t _ {k}, x [ k ], a) |.$$

When using the method proposed in [1], approximated control trajectories fluctuate frequently even though the trajectory of $\beta$ has no chattering issue. This is because the algorithm in [1] uses interpolation at every time step. Instead, our method chooses the nearest control input to obtain an admissible control that is less likely to display fluctuations as long as $\beta ( t ) \in \operatorname { C o n v } ( B ( x ( t ) , t ) )$ is not oscillating wildly.
