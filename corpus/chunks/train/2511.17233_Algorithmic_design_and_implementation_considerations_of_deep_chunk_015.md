In this section, we have chosen $u _ { \mathrm { m a x } } ^ { a } = 0 . 6$ and experimental results are demonstrated in Fig. 3 and Fig. 4 for 100 time steps. Fig. 3 demonstrates that the trajectory of tube-MPC and deep MPC are very close to each other due to the absence of enough learning authority. The term $\tilde { u } _ { t } = u _ { t } ^ { a } + h ( s _ { t } )$ in Fig. 4 does not converge to some neighbourhood of origin instead it follows the trajectory of $h ( s )$ with some offset. This phenomenon is due to the saturation of the learning control $\boldsymbol { u } _ { t } ^ { a }$ as shown in Fig. 4. This experiment concludes that learning is not happening due to the absence of enough learning authority and saturation of the training data $\boldsymbol { u } _ { t } ^ { a }$ resulting in the lack of relationship between $x _ { t }$ and $\boldsymbol { u } _ { t } ^ { a }$ . For practical purposes, we can use $u _ { \mathrm { m a x } } ^ { a }$ little larger or equal to the value obtained by using Algorithm 1, similar to the numerical experiment in [2].

We considered the same numerical experiment as in [2] but we chose smaller $u _ { \mathrm { m a x } } ^ { a }$ and demonstrated that deep MPC performs as tube-MPC (Fig. 3). We explained the reason of not getting advantage over tube-MPC in Fig. 4. The simulation result reveals that smaller $u _ { \mathrm { m a x } } ^ { a }$ leads to saturation of $\boldsymbol { u } _ { t } ^ { a }$ and therefore learning does not happen.
