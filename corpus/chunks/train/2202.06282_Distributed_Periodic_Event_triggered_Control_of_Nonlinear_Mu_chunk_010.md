# 3.2 Transmitted outputs

Due to the usage of a packet-based network, agent $A _ { i } ,$ $i \in \mathcal N$ , does not have continuous access to the output $y _ { m }$ of agent $\mathcal { A } _ { m } , m \in \mathcal { V } _ { i } ^ { \mathrm { i n } }$ . Instead, agent $\mathbf { \mathcal { A } } _ { i }$ only has an estimate $\widehat { y } _ { m } ^ { i }$ of $y _ { m } , m \in \mathcal { V } _ { i } ^ { \mathrm { i n } }$ , collected in the estimation vector $\widehat { y } _ { i } ^ { \mathrm { i n } } : = ( \widehat { y } _ { 1 } ^ { i } , \widehat { y } _ { 2 } ^ { i } , \ldots , \widehat { y } _ { N } ^ { i } ) \in \mathbb { R } ^ { n _ { y } }$ . For all m $\in \mathcal { N } \backslash \mathcal { V } _ { i } ^ { \mathrm { i n } }$ , $\widehat { y } _ { m } ^ { i }$ is not relevant and simply set to zero.

At discrete times $t _ { k } ^ { i } , \ k \in \mathbb { N } , \ i \in \mathcal { N } .$ that satisfy $0 \ : =$ $t _ { 0 } ^ { i } < t _ { 1 } ^ { i } < . . . ,$ the output $y _ { i }$ of agent $\mathbf { \mathcal { A } } _ { i }$ is broadcasted over the network to all (connected) agents $A _ { m } ,$ where $m \in \mathcal { V } _ { i } ^ { \mathrm { o u t } }$ with $\mathcal { V } _ { i } ^ { \mathrm { o u t } }$ the collection of all agents that receive information from agent $A _ { i }$ . Due to possible network delays, the estimate $\widehat { y } _ { i } ^ { m } , \ : m \in \mathcal { V } _ { i } ^ { \mathrm { o u t } }$ , is updated after a delay of $\Delta _ { k } ^ { i , m } \geqslant 0$ 2,m b  . Note that the delays are differe $\begin{array} { r } { \widehat { y } _ { i } ^ { m } , i \in \mathcal { N } , } \end{array}$ $\dot { m } \in \mathcal { V } _ { i } ^ { \mathrm { o u t } }$ , can be expressed as

$$\widehat {y} _ {i} ^ {m} ((t _ {k} ^ {i} + \Delta_ {k} ^ {i, m}) ^ {+}) = y _ {i} (t _ {k} ^ {i}). \tag {3}$$

In between transmissions, the estimate ${ \widehat { y } } _ { i } ^ { m }$ is generated by a zero-order-hold (ZOH) device, i.e.,

$$\dot {\hat {y}} _ {i} ^ {m} (t) = 0, \tag {4}$$

for all $t \in ( t _ { k } ^ { i } + \Delta _ { k } ^ { i , m } , t _ { k + 1 } ^ { i } + \Delta _ { k + 1 } ^ { i , m } )$ , with $i \in \mathcal { N } , m \in \mathcal { V } _ { i } ^ { \mathrm { o u t } }$ , $k \in \mathbb N .$ .

The transmission times $t _ { k } ^ { i }$ are determined by an event generator or triggering mechanism. Inspired by Girard (2015) and Dolk et al. (2017), we consider dynamic event triggering rules, where an auxiliary variable $\eta _ { i } ~ \in ~ \mathbb { R } _ { \geqslant 0 } .$ , $i \in \mathcal { N }$ , whose dynamics are designed in the sequel, is used to determine the transmission times $t _ { k } ^ { i } , k \in \mathbb { N }$ , see Section 3.4.
