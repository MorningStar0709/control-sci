# C. Gradient-based Optimization Approach

Having defined both the high-fidelity and low-fidelity MPC formulations, the next step is to construct the inverse MPC as described in (2). The goal is to derive the “optimal” malicious parameters to inject into the firmware, such that the compromised low-fidelity MPC yields a solution as close as possible to the adversarial control sequence generated by the high-fidelity adversarial MPC. To achieve this, we define the discrepancy loss function as $l ( u _ { t } ^ { * } ( \theta ) , u _ { t } ^ { \mathrm { a d v } } )$ = $\lVert u _ { t } ^ { * } ( \theta ) - u _ { t } ^ { \mathrm { a d v } } \rVert ^ { 2 }$ .

Given this bi-level optimization problem, there are several methods to solve it, as discussed in [12]. One way is to adopt a computationally efficient, gradient-based approach:

$$\theta_ {i + 1} = \theta_ {i} - \alpha \frac {\mathrm{d} L}{\mathrm{d} \theta} \Big | _ {\theta_ {i}},$$

where i denotes the iteration index, α is the learning rate, and $\begin{array} { r } { \frac { \mathrm { d } L } { \mathrm { d } \theta } \bigg \vert _ { \theta _ { i } } } \end{array}$ is the gradient value of the loss w.r.t. θ evaluated θi at $\theta _ { i }$ . The gradient term $\begin{array} { r } { \frac { \mathrm { d } L } { \mathrm { d } \theta } \bigg \vert _ { \theta _ { j } } } \end{array}$ can be computed based on the θi Pontryagin Differential Principle (PDP) mindset proposed in [13].
