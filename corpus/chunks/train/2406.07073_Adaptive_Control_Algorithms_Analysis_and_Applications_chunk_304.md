# 7.3.2 Tracking

In the ideal case we would like to perfectly follow a desired trajectory $y ^ { \star } ( t + d + 1 )$ , known $d + 1$ steps ahead which is stored in the computer or generated from the reference signal via a tracking reference model, i.e.,

$$H _ {m} (q ^ {- 1}) = \frac {B _ {m} (q ^ {- 1})}{A _ {m} (q ^ {- 1})} \tag {7.32}$$

with

$$B _ {m} \left(q ^ {- 1}\right) = b _ {m 0} + b _ {m 1} q ^ {- 1} + \dots + b _ {m n _ {B m}} q ^ {- n _ {B m}} \tag {7.33}A _ {m} \left(q ^ {- 1}\right) = 1 + a _ {m 1} q ^ {- 1} + \dots + a _ {m n _ {A m}} q ^ {- n _ {A m}} \tag {7.34}$$

Often, this tracking reference model is determined from the desired tracking performances (rise time, settling time, overshoot) by selecting either an appropriate normalized second order system (defined by $\omega _ { 0 } , \zeta )$ or sometimes a cascade of two second orders. Once a continuous-time reference model is selected, one gets by discretization the discrete-time tracking reference model. Note, however, that the discrete-time tracking reference model can be directly defined in discrete time (see Landau 1990b).
