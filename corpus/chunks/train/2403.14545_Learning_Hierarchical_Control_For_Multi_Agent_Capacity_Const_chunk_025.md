The controller searches at each time step t for a feasible $N \cdot$ - step trajectory from the current state $\boldsymbol { x } _ { t } ^ { r }$ to a state $x _ { t + N | t } =$ in the safe set $\left( c _ { t + N | t } , \chi _ { t + N | t } \right)$ $S _ { i , j } ^ { L , r }$ , where in two ways: $x _ { t + N | t }$ relates to a state $x _ { t \star } = ( \bar { c } , \bar { \chi } )$

1) the non-capacity states $\chi _ { t + N | t }$ are the same as those of state $x _ { t \star } ~ ( 3 3 \mathrm { g } )$ , and   
2) the capacity states $c _ { t + N | t }$ have depleted less since the beginning of the edge trajectory (where capacity states had value $c _ { k } ^ { r } )$ than those of $\boldsymbol { x } _ { t \star } \in \mathcal { S } _ { i , j } ^ { L , r }$ (33f).

The objective function minimizes the sum of two terms: the stage cost (16) accumulated over the planned N-step trajectory, and the remaining stage cost accumulated on the trajectory from $x _ { t \star }$ back to the depot in a previous iteration. At time step $t ,$ the first planned input is applied and a new optimal control problem (33) is solved at time step t + 1.

It follows from [26] and construction of (16) and (33) that there exists a time step $t ^ { \star }$ while the low-level policy (33) is being applied such that $x _ { t ^ { \star } + N | t ^ { \star } } \in \mathcal { N } _ { j }$ , i.e. the planned $N -$ step path reaches the goal node $v _ { j }$ .
