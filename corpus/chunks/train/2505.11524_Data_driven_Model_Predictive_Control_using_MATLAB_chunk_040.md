Lemma 1 (Fundamental lemma [17]). Suppose $\{ \mathbf { u } _ { 0 } , \mathbf { u } _ { 1 } , \dotsc , \mathbf { u } _ { \mathrm { D - 1 } } \}$ is persistently exciting of order $\mathrm { N } + \mathrm { M } + n$ . Then for every trajectory $\left( \mathbf { Y } _ { k + 1 } , \mathbf { U } _ { k } \right)$ of the system there exists a solution $\mathbf { v } _ { k } ^ { \star } \in \mathbb { R } ^ { \mathrm { H } }$ satisfies:

$$
\left[ \begin{array}{l} \mathbf {U} _ {\mathrm{p}} \\ \mathbf {Y} _ {\mathrm{p}} \\ \mathbf {U} _ {\mathrm{f}} \\ \mathbf {Y} _ {\mathrm{f}} \end{array} \right] \mathbf {v} _ {k} = \left[ \begin{array}{l} \mathbf {U} _ {\mathrm{p1}} \\ \mathbf {Y} _ {\mathrm{p1}} \\ \mathbf {U} _ {k} \\ \mathbf {Y} _ {k + 1} \end{array} \right]. \tag {59}
$$

The Eq. (59) given by fundamental lemma can be considered as data-driven equality constraint which can be used instead of model-based equality constraints. The state space model in the MPC optimization problem in Eq. (12) or the model-based equality constraint in Eq. (14) can be replaced by a data-dependent linear equality constraint defined with Hankel matrices as in Eq. (59). By doing so, DeePC effectively bypasses the need for an intermediate system identification step, offering a powerful and flexible framework for predictive control in both linear and certain nonlinear systems. In this context, DeePC can be regarded as a promising method within the broader scope of adaptive control. The optimization problem for DeePC is defined as:

$$\inf _ {\mathbf {U} _ {k}, \mathbf {Y} _ {k + 1}, \mathbf {v} _ {k}} \left[ \mathbf {Y} _ {r} - \mathbf {Y} _ {k + 1} \right] ^ {\top} \mathbf {Q} _ {\mathbf {Y}} \left[ \mathbf {Y} _ {r} - \mathbf {Y} _ {k + 1} \right] + \mathbf {U} _ {k} ^ {\top} \mathbf {R} _ {\mathbf {U}} \mathbf {U} _ {k} + \alpha L (\mathbf {v} _ {k})$$

$\mathrm { s u b j e c t ~ t o ~ } \mathbf { Y } _ { k + 1 } \in \mathbb { Y } ^ { \mathrm { N } } , \mathbf { U } _ { k } \in \mathbb { U } ^ { \mathrm { N } }$

$$
\left[ \begin{array}{l} \mathbf {U} _ {\mathrm{p}} \\ \mathbf {Y} _ {\mathrm{p}} \\ \mathbf {U} _ {\mathrm{f}} \\ \mathbf {Y} _ {\mathrm{f}} \end{array} \right] \mathbf {v} _ {k} = \left[ \begin{array}{l} \mathbf {U} _ {\mathrm{p} 1} \\ \mathbf {Y} _ {\mathrm{p} 1} \\ \mathbf {U} _ {k} \\ \mathbf {Y} _ {k + 1} \end{array} \right] \tag {60}
$$
