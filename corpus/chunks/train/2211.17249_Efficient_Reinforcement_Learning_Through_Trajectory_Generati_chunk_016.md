# 3.3. Algorithm

By Theorem 2, using historical data, given $\pmb { x } ( 0 )$ and perturbations on actions ${ \pmb w } ( 0 ) , \dots , { \pmb w } ( T - 1 )$ , a trajectory can be generated as if it comes from sampling the system with the current control policy. The details of the algorithm is illustrated in Algorithm 1. We also use REINFORCE policy gradient (Sutton and Barto, 2018) in Appendix 7.2 as an example to show how to use the trajectory generation algorithm in RL methods. The key benefit of Algorithm 1 is that it is adaptive to the updates of parameter θ and ${ \pmb w } ( t )$ for exploration. In particular, ${ \pmb w } ( t )$ can be sampled from any distribution D, making it versatile for different applications.

Algorithm 1 Trajectory generation for state-feedback control   
Data collection: Collect historic measurement of the system and stack each T-length input-output trajectory as Hankel matrix H shown in (5) until $\text{rank}(\mathcal{H}) = n + Tm$ Data generation: Input : Hankel matrix H, weights $\theta$ and the distribution D for the control policy, the batchsize Q (number of the generated trajectories), the distribution $S_x$ of the initial states $^{1}$ Function TrajectoryGen( $H, \theta, D, Q, S_x$ ):
Plug in $\theta$ to compute $G_\theta = [\mathcal{H}_u - (\mathbf{I}_T \otimes \boldsymbol{\theta}) \mathcal{H}_x; \mathcal{H}_x^0]$ for i = 1 to Q do
    Sample $x_i(0)$ from $S_x$ . Sample $\{\boldsymbol{w}_i(0), \cdots, \boldsymbol{w}_i(T-1)\}$ from D.
    Compute the coefficient $g_i^* = G_\theta^\top (G_\theta G_\theta^\top)^{-1} [w_i(0); \cdots; w_i(T-1); x_i(0)]$ .
    Generate the i-th trajectory $\tau_i := [\tilde{u}_i(0); \cdots; \tilde{u}_i(T-1); \tilde{x}_i(0); \cdots; \tilde{x}_i(T-1);] = \mathcal{H}g_i^*$ .
end
return $[\tau_1, \cdots, \tau_Q]$
