# A. Buffering Mechanism

The proposed buffering mechanism is implemented as a part of the actuator node at the receiving end of the feedback channel at the plant side, see Fig. 1. Its effect is defined such, that, if a data packet arrives at the plant side, the buffer adds an additional delay $\tau _ { k } ^ { b }$ before forwarding the control signal to the ZOH, i.e. the overall delay is reformulated to $\tau _ { k } = \tau _ { k } ^ { s c } + \tau _ { k } ^ { c } + \tau _ { k } ^ { c a } + \tau _ { k } ^ { b }$ . The delay $\tau _ { k } ^ { b }$ is calculated based on the timestamp attached by the sensor, so that $\begin{array} { r } { \tau _ { k } \ = \ \left\lceil \frac { \tau _ { k } ^ { s c } + \tau _ { k } ^ { c } + \tau _ { k } ^ { c a } } { T _ { d } } \right\rceil \cdot T _ { d } \ = \ q _ { k } T _ { d } } \end{array}$ T τ s ck + τ ck + τ c ak holds. The buffer delay is therefore given as $\tau _ { k } ^ { b } = q _ { k } T _ { d } - ( \tau _ { k } ^ { s c } + \tau _ { k } ^ { c } + \tau _ { k } ^ { c a } ) < T _ { d }$ . This ensures that the control signal is updated at $t = k T _ { d }$ and that there is only one signal acting on the plant during one sampling period. In the case when more than one data packet arrive during one sampling period, the most recently received control signal is applied, whereas in the case when no new signal is received, the previously applied $\mathbf { \Delta } \mathbf { u } _ { k }$ is kept active. Even though it is still unknown when a specific control signal will be applied to the system, the minimal time duration for which an applied control input is active is defined explicitly. The NCS can be therefore described by a simplified, discretetime model

$$\boldsymbol {x} _ {k + 1} = \boldsymbol {A} _ {d} \boldsymbol {x} _ {k} + \boldsymbol {B} _ {d} \boldsymbol {u} _ {k - q _ {k}} \tag {4}$$

with $\begin{array} { r } { \pmb { B } _ { d } = \int _ { 0 } ^ { T _ { d } } e ^ { \pmb { A } _ { c } ( T _ { d } - s ) } \pmb { B } _ { c } d s = \left[ \pmb { b } _ { 1 } \quad \pmb { b } _ { 2 } \quad . . . \quad \pmb { b } _ { m } \right] } \end{array}$ a n d $q _ { k } \in \{ 1 , 2 , \dotsc , \bar { \delta } \}$ being the unknown input delay which depends on the network imperfections, see Assumptions 2 and 3. A model suited for stability analysis is obtained by defining the following lifted-state vector

$$
\boldsymbol {\xi} _ {k} = \left[ \begin{array}{l l l l l l} \boldsymbol {x} _ {k} ^ {T} & u _ {1, k - 1} & u _ {1, k - 2} & \dots & u _ {1, k - \delta} & \dots \end{array} \right] _ {T}. \tag {5}

\left. \begin{array}{c c c c} u _ {m, k - 1} & u _ {m, k - 2} & \ldots & u _ {m, k - \delta} \end{array} \right] ^ {T}
