# C. Controllers Design

The weights of the quadratic criterion used in designing the controllers were appropriately selected based on the UAV’s behavior and surrounding environment as

$$
\boldsymbol {Q} _ {y} ^ {h} = \left[ \begin{array}{c c} 0. 1 6 & 0 \\ 0 & 0. 0 4 \end{array} \right],   \boldsymbol {Q} _ {z} ^ {h} = \left[ \begin{array}{c c} 0. 6 4 & 0 \\ 0 & 0. 0 4 \end{array} \right],
\boldsymbol {R} _ {y} ^ {h} = \frac {1}{2} \cdot 0. 1 5 \cdot \frac {4}{0 . 0 3} \cdot \sin 3 0 ^ {\circ} = 5, \boldsymbol {R} _ {z} ^ {h} = 5 ^ {- 2} = 0. 0 4,$$

where $\pmb { Q } _ { \mathrm { y } } ^ { h } , \pmb { R } _ { \mathrm { y } } ^ { h } , \pmb { Q } _ { \mathrm { z } } ^ { h }$ , and $\pmb { R } _ { z } ^ { h }$ are the weights utilized in MPC, IC, and eIC for position control in $\vec { y }$ and ⃗z-axis.

For trajectory tracking, IC employs the LQR $\pmb { u } ^ { l }$ with the following weight configuration:

$$
\boldsymbol {Q} _ {y} ^ {l} = \left[ \begin{array}{c c} 0. 2 5 & 0 \\ 0 & 0. 0 4 \end{array} \right], \boldsymbol {Q} _ {z} ^ {l} = \left[ \begin{array}{c c} 0. 6 4 & 0 \\ 0 & 0. 0 4 \end{array} \right],
\boldsymbol {R} _ {y} ^ {l} = 1 0 \cdot \boldsymbol {R} _ {y} ^ {h} = 5 0, \boldsymbol {R} _ {z} ^ {l} = 1 0 \cdot \boldsymbol {R} _ {z} ^ {h} = 0. 4.$$

The setpoint LQR $\pmb { u } ^ { m }$ uses the following weighting matrices

$$\boldsymbol {Q} _ {y} ^ {m} = \boldsymbol {Q} _ {y} ^ {l}, \boldsymbol {Q} _ {z} ^ {m} = \boldsymbol {Q} _ {y} ^ {l}, \boldsymbol {R} _ {y} ^ {m} = \boldsymbol {R} _ {y} ^ {h}, \boldsymbol {R} _ {z} ^ {m} = \boldsymbol {R} _ {z} ^ {h}. \tag {12}$$

The predictive horizon was set to 8s as in [1], resulting in $N =$ 800 steps for the $T _ { s } = 0 . 0 1 { \mathrm { s } }$ discrete model. However, such a long horizon is not feasible for real-time MPC implementation. To address the issue, we employed the move blocking technique [11], which fixes the control variable values $\pmb { u } _ { k } ^ { k + N }$ for multiple time steps, reducing the $\mathrm { Q P s }$ degrees of freedom. To further reduce complexity, the prediction for $T _ { s } = 0 . 0 1 { \mathrm { s } }$ is performed only in the first step and afterward, the model with $T _ { s } = 0 . 2 \mathrm { s }$ is considered. The complexity of the problem was reduced by 95%. This version of the MPC will be referred to hereafter as MPCMB.

A basic discrete-time PID controller is implemented in Python with experimentally obtained parameters $K _ { p } = 0 . 3$ , $K _ { d } = 0 . 0 0 3$ , and $K _ { i } = 0 . 0 0 0 1$ , and with a period $T _ { s \_ a t t } = 0 . 0 0 1 { \mathrm { s } }$ The PID simulates the attitude control behavior in a scenario with the planar UAV model, which would otherwise be done by the autopilot.
