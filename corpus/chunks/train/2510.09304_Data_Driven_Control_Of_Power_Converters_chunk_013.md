# • Step 4: Control tuning

First, the virtual reference signal r(t) is constructed so to satisfy $V _ { o u t } ( t ) ~ = ~ M ( z ) \cdot \breve { r ( t ) }$ . It corresponds to the reference that would have been sent to the desired closedloop model in order to get the training output $V _ { o u t } ( t )$ . The corresponding virtual tracking error $e ( t ) = r ( t ) - \dot { V } _ { o u t } ( t )$

is then computed. The ideal controller $K ^ { \star }$ is then the one that gives the training signal $d ( t )$ when it is fed with the virtual error $e ( t )$ : this is how the control tuning problem is recast as controller identification.

The controller parameters $\theta = [ K _ { p } K _ { I } ] ^ { T }$ are then found by minimizing the following criterion

$$
\begin{array}{l} J _ {V R F T} = \frac {1}{N} \sum_ {t = 1} ^ {N} (d (t) - K (z; \theta) \cdot e (t)) ^ {2} \tag {7} \\ = \frac {1}{N} \sum_ {t = 1} ^ {N} (d (t) - \Phi^ {T} (t) \cdot \theta) ^ {2} \\ \end{array}
$$

Knowing that $K ( z , \theta , ) = \beta ^ { T } ( z ) \theta$ , the regressor is $\Phi ( t ) = $ $\beta ( z ) e ( t )$ . This is a least squares problem that can be solved easily. The solution $\theta ^ { \star }$ is given by:

$$\theta^ {\star} = [ \sum_ {t = 1} ^ {N} \Phi (t) \cdot \Phi (t) ] ^ {- 1} \sum_ {t = 1} ^ {N} \Phi (t) \cdot d (t) \tag {8}$$

In the end, the following parameters are estimated :

$$\theta^ {\star} = \left[ K _ {p} ^ {\star} K _ {I} ^ {\star} \right] ^ {T} = [ 0. 0 0 3 1 0. 0 0 6 5 ] ^ {T}$$

After discretization, the PI controller implemented in the Simulink model is the following :

$$K (s) = 0. 0 0 3 1 + 0. 0 0 6 5 \cdot \frac {T _ {s}}{s} \tag {9}$$

While one can see that the VRFT controller performs better than the ZN one with a faster response, a significant undershoot is observed for both controllers.
