# A. Finding a trajectory with MPC

We consider a discrete-time optimal control problem of finding the optimal sequence of control inputs $u _ { 0 } , u _ { 1 } , \ldots , u _ { N - 1 } \in \mathbb { R } ^ { m }$ , driving the states $x _ { 0 } , x _ { 1 } , \dotsc , x _ { N } \in$ $\mathbb { R } ^ { n }$ of a dynamical system $x _ { k + 1 } ~ = ~ f ( x _ { k } , u _ { k } )$ for $k \_ =$ $0 , 1 , \ldots , N - 1$ , where N is the horizon length.

Denote by $\mathbf { u } _ { 0 : N - 1 } : = \{ u _ { 0 } , u _ { 1 } , \dots , u _ { N - 1 } \}$ the sequence of control inputs and by $\mathbf { x } _ { 0 : N } \ : = \ \{ x _ { 0 } , x _ { 1 } , . . . , x _ { N } \}$ the corresponding sequence of states. Let $r ( x _ { k } )$ be the reward at state $x _ { k }$ . Then the (non-linear) optimal control problem is given by

$$
\begin{array}{l l} \min _ {\mathbf {x} _ {0: N}, \mathbf {u} _ {0: N - 1}} & J (\mathbf {x} _ {0: N}, \mathbf {u} _ {0: N - 1}), \\ \text { s.t. } \end{array} \tag {2a}
x _ {k + 1} = f (x _ {k}, u _ {k}) \quad \forall k = 0, \dots , N - 1, \tag {2b}x _ {k} \in \mathbb {X} \quad \forall k = 0, \dots , N, \tag {2c}u _ {k} \in \mathbb {U} \quad \forall k = 0, \dots , N - 1, \tag {2d}x _ {0} = x (0). \tag {2e}$$

The objective function $J ( \mathbf { x } _ { 0 : N } , \mathbf { u } _ { 0 : N - 1 } )$ contains the control input costs and the state rewards:

$$J (\mathbf {x} _ {0: N}, \mathbf {u} _ {0: N - 1}) = c _ {1} \sum_ {k = 0} ^ {N - 1} u _ {k} ^ {\top} R u _ {k} - c _ {2} \sum_ {k = 0} ^ {N} r (x _ {k}). \tag {3}$$

For the control input costs we choose a quadratic formulation, defined by a positive definite cost matrix $R \in \mathbb { R } ^ { m \times m }$ . The second term in the objective function maximizes the reward along the path, approximating the integral in (1). Both terms are weighted by the (positive) parameters $c _ { 1 }$ and $c _ { 2 } ,$ respectively.

The constraints (2c) and (2d) keep the states and inputs in the set of admissible states (the region of the map) X and in the set of admissible inputs (accelerations) U, respectively. The constraint (2e) sets the initial state of the agent.

To prevent the agent from just collecting the same reward multiple times, we introduce additional coverage constraints

$$\rho (x _ {i}, x _ {j}) \geq V - \epsilon_ {i}, \quad i = 1, \dots , N, j = 0, \dots , i - 1 \tag {4}$$
