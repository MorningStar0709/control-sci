# 4.1 Algorithm Summary

The proposed method is summarized by Algorithm 1. The algorithm is written as on-policy learning, meaning that the policy updates are based on data collected while using the current policy. For each policy update, data is collected using M different set points, where $M > 1$ , in order to stabilize the training process. The method is also possible to combine with off-policy RL methods, where also data collected while using older polices can be utilized in each update.

The PPO algorithm is chosen in the paper to optimize the residual policy, this will be denoted as ResidualPPO throughout the rest of the paper. The clipped surrogate objective in PPO, mentioned in Section 3.3, improves the training when using a prior policy to track various reference signals. For each update, the training data only contains a limited number of reference signal samples and thus it may be better not to move away from the old policy too fast.

Algorithm 1 Residual Multi-goal Reinforcement Learning   
Input: Policy $\pi_{\theta}$ , prior controller $g^{o}$ , goal space G
1: for iteration=1,2,... do
2: Start collecting data with M different goals
3: for $m=1,2,\ldots,M$ do
4: Sample $y_{m}^{ref}$ from G uniformly
5: Sample initial state of dynamic system $x_{0}$ (for real-world systems, just start from whatever state the system is in)
6: for $t=0\ldots T-1$ , do
7: Get prior input $u_{t}^{o}=g^{o}(x_{t},y_{m}^{\mathrm{ref}})$ 8: Sample input from policy distribution $\pi_{\theta}$ by $\delta_{t}^{u}\sim\mathcal{N}(g_{\theta}(x_{t},y_{m}^{\mathrm{ref}}),\sigma_{\theta}(x_{t},y_{m}^{\mathrm{ref}}))$ 9: Apply the input $u_{t}=u_{t}^{o}+\delta_{t}^{u}$ 10: Get next state $x_{t}$ from dynamics and reward signal $r_{t}$ from reward function
11: Store $(x_{t},y_{m}^{\mathrm{ref}},u_{t},x_{t+1},r_{t})$ 12: end for
13: end for
14: Optimize $\theta$ using RL algorithm, e.g., PPO.
15: end for
