Remark 2: It is important to note that once the risky set $\mathcal { C } _ { \mathrm { R } }$ is reached for the first time, denoted as $t _ { 1 }$ , the safeguarding control mechanism becomes active for $t \geq t _ { 1 }$ , even if the system trajectory does not intersect $\mathcal { C } _ { \mathrm { R } }$ again. In other words, the initial strategy is deactivated for $t ~ > ~ t _ { 1 }$ by setting $\bar { h } = + \infty$ , resulting in a single switch from the conventional sliding mode controller to the proposed safetycritical controller. Without resetting the risky set to the entire state space for $t > t _ { 1 }$ , the finite time convergence of the system state to the sliding manifold cannot be guaranteed, as there could be an infinite number of switches between $\mathcal { C } _ { \mathrm { R } }$ and $\bar { \mathcal { C } } _ { \mathrm { R } }$ . Under the second strategy, there is a possibility of only one additional switch from the proposed controller back to the conventional sliding mode controller.
