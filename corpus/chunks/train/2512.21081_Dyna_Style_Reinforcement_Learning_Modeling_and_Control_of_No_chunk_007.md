Algorithm 1 Dyna-Style SINDy-TD3 Framework   
1: Input: Initial dataset $\mathcal{D}=\{(x_{t},u_{t},x_{t+1})\}$ , RL hyperparameters $\gamma,\tau,\sigma,N$ , reward function $r(x,u)$ 2: Output: Trained control policy $\pi_{\phi}$ 3: Data Collection: Collect initial real-world trajectories $D_{real}$ from the physical system using an exploratory or random policy.
4: Model Learning: Train a SINDy model $f_{\mathrm{SINDy}}(x,u)\approx\dot{x}$ using $D_{real}$ .
5: Initialization: Initialize replay buffer $B\leftarrow D_{real}$ , actor network $\pi_{\phi}$ , and critic networks $Q_{\theta_{1}},Q_{\theta_{2}}$ .
6: for each episode do
7: Real Interaction: Interact with the real environment using the current policy $\pi_{\phi}$ to collect transitions $(x_{t},u_{t},r_{t},x_{t+1})$ .
8: Add collected real transitions to the replay buffer B.
9: Policy Learning: Sample mini-batches from B and update the actor and critics using the TD3 algorithm.
10: Planning: After every N episodes, update the SINDy model using recent real transitions from B, simulate synthetic rollouts using the updated model to generate model-based transitions $D_{model}$ , and add them to the replay buffer:

$$\mathcal {B} \leftarrow \mathcal {B} \cup \mathcal {D} _ {\mathrm{model}}$$

11: end for   
12: Deployment: Deploy the final trained policy $\pi _ { \phi }$ on the real system for control and tracking.
