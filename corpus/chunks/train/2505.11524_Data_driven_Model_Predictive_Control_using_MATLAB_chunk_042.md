# 3.4.1 DeePC: Numerical example

To demonstrate the DeePC approach, the same example presented for SPC in Section 3.3.1 is considered, with the system parameters defined as in Equation (58). The training data used to construct the Hankel matrices is identical, with the parameters set to $\mathrm { \Delta N = 3 0 , M = 2 0 }$ , and $\mathrm { H } = 2 0 \bar { 0 }$ . Based on the resulting input-output Hankel matrices, the DeePC algorithm is implemented as outlined in Algorithm 3. The simulation parameters are selected as follows: $\mathrm { N _ { T } = 2 0 0 , N } \mathrm { \bar { = } 3 0 , \mathbf { Q _ { y } } = 5 , \mathbf { R _ { u } } = 0 . 1 }$ , and $\alpha = 1$ . The input constraints are defined by $u _ { \mathrm { m i n } } = - 5$ and $u _ { \mathrm { m a x } } = 5$ . Fig. 4(a) illustrates the input-output training data used for constructing the Hankel matrices. The simulation results for output tracking using DeePC are presented in Fig. 4(b), demonstrating that the system output effectively tracks the desired reference signal. The corresponding control input is shown in Fig. 4(c).
