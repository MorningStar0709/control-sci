Remark 2. There is an exploration-exploitation trade-off in selecting $f ( n )$ on the two sides of $\mu = c / R$ . When admitting is optimal, we want $f ( n )$ to increase to infinity as slow as possible. Also, based on the proof of Theorem 3, for our current bound, we cannot take $f ( n )$ to grow exponentially fast since its exponent needs to depend on unknown $\mu$ to ensure constant regret. Conversely, when blocking all arrivals is optimal, we need $f ( n )$ to converge to infinity as fast as possible. As the learning algorithm needs to be agnostic about the parameter regime, $f ( n ) = \exp \left( n ^ { 1 - \epsilon } \right)$ is a good choice: it ensures constant regret in one regime and polynomial regret in log(n) in the other.
