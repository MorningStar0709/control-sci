Constraints (9b)-(9d) impose the typical MPC constraints, namely, the initial state, system dynamics and system constraints. Constraint (9e) imposes that the predicted state $x ^ { N }$ reaches the value of the artificial harmonic reference at $k = N .$ . The equality constraints (9f) impose the satisfaction of the system dynamics (2) on the artificial harmonic reference, as shown in Proposition 1, whereas the second order cone constraints (9g) impose the strict satisfaction of the system constraints (3) on the artificial harmonic reference, as shown in Proposition 2. The satisfaction of (9f) and (9g) implies that the artificial harmonic reference is a strictly admissible harmonic signal of system (2) subject to (3), where strict satisfaction of the constraints is attained due to the inclusion of the scalar $\sigma > 0$ in (2), as stated in Corollary 1.

The cost function of the HMPC formulation penalizes, on one hand, the discrepancy between the predicted states $x ^ { k }$ and inputs $u ^ { k }$ with the values $x _ { h } ^ { k }$ and $\bar { u } _ { h } ^ { k }$ of the artificial harmonic reference at prediction time $k ,$ respectively, and on the other hand, the discrepancy between the parameters $\left( \mathbf { x } _ { h } , \mathbf { u } _ { h } \right)$ with $( \mathbf { x } _ { r } ( t ) , \mathbf { u } _ { r } ( t ) )$ . The effect is that the artificial harmonic reference will tend towards the reference, while in turn the predicted states will tend towards the artificial harmonic reference. Let $\mathbf { x } ^ { * } , \ \mathbf { u } ^ { * } , \ \mathbf { x } _ { h } ^ { * }$ and $\mathbf { u } _ { h } ^ { * }$ be the optimal solution of (9). The control input $u ( t )$ is taken as the first move in the sequence of optimal inputs $\mathbf { u } ^ { * }$ . We note that (9) can recover the classical local optimality of MPC for tracking (see [8, Property 1]) if, for instance, linear penalization terms are added to $V _ { h } .$ , i.e, $\alpha _ { e } \| x _ { e } - x _ { r e } \| , \alpha _ { s } \| x _ { s } - x _ { r s } ( t ) \| , \alpha _ { c } \| x _ { c } -$ $x _ { r c } ( t ) \| , \beta _ { e } \| u _ { e } - u _ { r e } \| , \beta _ { s } \| u _ { s } - u _ { r s } ( t ) \| , \beta _ { c } \| u _ { c } - u _ { r c } ( t ) \|$ , with $\alpha _ { \{ e , s , c \} } , \beta _ { \{ e , s , c \} } \geq 0$ large enough. However, in this article we focus on the previously shown quadratic offset cost $V _ { h }$ , since it results in a much simpler to solve optimization problem; a typicall choice in linear MPC for tracking for this reason [25].
