Algorithm 2 Data Generation for Noisy Data w/ Control   
Input: Number of Initial Conditions, time step dt, final time $t_{f}$ , data points dp, state length $n_{x}$ , control input length m.
Method
1: for num in range(number of Initial Conditions) do
2: Generate random I.C with noise $x_{init} = 2 * torch.Tensor(2).uniform_{-}(-1,1) + (0.0001 ** 0.5) * np.random.normal(0,1,2)$ 3: Declare the time vector for each I.C $time = torch.linspace(0, t_{f}, dp)$ 4: for t in time do
5: Generate random control input $u_{k} = torch.Tensor(1).uniform_{-}(-1,1)$ 6: Save the current state in the X data array
7: Integrate the current state using the dynamics and RK4 functions. Add noise to new state $x_{k+1} = rk4(pendfunc, t, x_{k}, u_{k}, dt) + (0.0001 ** 0.5) * np.random.normal(0,1,2)$ 8: Save the new state in the Y data array.
9: end for
10: Stack the solutions, X,Y,u from the I.C. into 3D array that holds every trajectory for each I.C. $data_{i} = torch.vstack([data_{i}, new_{i}[None, :])$ 11: end for
Output: Data matrices, $data_{x}$ , $data_{y}$ , $data_{u}$ , input parameters.

The training data includes a random control effort of -1 or 1 to simulate the control effort that would be normally applied when deployed. This randomly excites the system and enables the extraction of the control input matrix B. Additionally, to check the robustness of the model, random Gaussian noise is added to each state to simulate sensor measurement noise. This is achieved by adding noise with the same standard deviation to each state at every integration time step with $\mu = 0 . 0 1 5 8$ and $\sigma = 0 . 0 0 9 1 3$ . See Algorithm 2.
