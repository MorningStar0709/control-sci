# B. Safety

We require that the safe SMPC algorithm described in Section III-A is safe. Based on Definition 1, this requirement demands that all constraints are met at all time steps, which we show in the following.

Theorem 2 (Safety): Let Assumptions 1 and 2 hold and let the system input $\mathbf { \pmb { u } } _ { t }$ be determined based on the proposed safe SMPC algorithm in Section III-A. Then, for a safe initial state $\scriptstyle { \pmb x } _ { 0 }$ , safety according to Definition 1 is guaranteed for $t > 0$ .

Proof: The proof is based on Theorem 1; hence, it is guaranteed that one of the two modes is applicable at each time step t. In the stochastic mode, $\mathbf { } u _ { t } \in \mathcal { U }$ and (9) ensures that $\mathbf { \boldsymbol { x } } _ { t } \in \mathcal { X }$ for all $\mathbf { \boldsymbol { w } } _ { t } \in \mathcal { W } .$ . The backup mode guarantees, by design, that $\mathbf { } u _ { t } \in \mathcal { U }$ and $\mathbf { \boldsymbol { x } } _ { t } \in \mathcal { X }$ for all $\boldsymbol { w } _ { t } \in \mathcal { W }$ . Hence, in both modes it is guaranteed that $\mathbf { \boldsymbol { x } } _ { t } ~ \in ~ \mathcal { X }$ and $\mathbf { \boldsymbol { u } } _ { t } \in \mathcal { U }$ , which holds for all $t \in \mathbb { N }$ as the proposed SMPC algorithm is recursively feasible.

As shown, safety is ensured by the backup predictive controller and (9), despite SMPC allowing for constraint violations in the open-loop prediction.
