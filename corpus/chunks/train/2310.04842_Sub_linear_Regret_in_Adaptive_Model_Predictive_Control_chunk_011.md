# 3.3. The tube MPC problem

Let $\pmb { v } _ { t } = \{ v _ { 0 | t } , \ldots , v _ { N - 1 | t } \}$ and $\pmb { \alpha } _ { t } = \{ \alpha _ { 0 | t } , \ldots , \alpha _ { N | t } \}$ . The resulting tube MPC problem, denoted as $\mathcal { P } _ { N } ( x _ { t } , \theta )$ for any parameter θ is:

$$\underset {\boldsymbol {v} _ {t}, \boldsymbol {\alpha} _ {t}} {\text { minimize }} \sum_ {k = 0} ^ {N - 1} \left(x _ {k | t} ^ {\top} Q x _ {k | t} + v _ {k | t} ^ {\top} R v _ {k | t}\right) + x _ {N | t} ^ {\top} P (\theta) x _ {N | t}$$

subject to, for all $j = 1 , \ldots , m$ , and $k = 0 , \ldots , N - 1$ :

initial constraints:

$$x _ {0 \mid t} = x _ {t}, \tag {6}T x _ {0 \mid t} \leq \alpha_ {0 \mid t}, \tag {7}$$

system constraints,

$$x _ {k + 1 | t} = \Phi (\theta) x _ {k | t} + B (\theta) v _ {k | t}, \tag {8}$$

tube constraints,

$$H _ {t} ^ {(j)} \alpha_ {k | t} + T B (\theta^ {(j)}) v _ {k | t} + \bar {w} _ {t} \leq \alpha_ {k + 1 | t}, \tag {9}H _ {c} \alpha_ {k | t} + G v _ {k | t} + \bar {\zeta} _ {t} \leq \mathbf {1}, \tag {10}$$

terminal constraints:

$$H _ {t} ^ {(j)} \alpha_ {N | t} + \bar {w} _ {t} \leq \alpha_ {N | t}, \tag {11}H _ {c} \alpha_ {N | t} + \bar {\zeta} _ {t} \leq \mathbf {1}. \tag {12}$$

where $P ( \theta )$ is obtained by solving the Lyapunov equation:

$$P (\theta) - \Phi (\theta) ^ {\top} P (\theta) \Phi (\theta) = Q + K ^ {\top} R K \tag {13}$$

Remark 3.3. The oracle tube-MPC problem $\mathcal { P } _ { N } ( x _ { t } , \theta ^ { \star } )$ is similarly defined by setting $\theta = \theta ^ { \star } , \bar { w } _ { t } = \bar { w } _ { t } ^ { \star }$ (with $[ \bar { w } _ { t } ^ { \star } ] _ { i } =$ $\operatorname* { m a x } _ { w \in \mathcal { W } } [ T w ] _ { i }$ , for $i \in \{ 1 , \ldots , d _ { \alpha } \} )$ , and $m = 1$ . The later can be achieved by Algorithm 1 by removing redundant constraints at every iteration by solving a linear program or one of the methods outlined in (Paulraj et al., 2010).

The feasibility of the above problem depends on whether $\theta ^ { \star } \in \Theta _ { t }$ , i.e., on the event G. However, for the sake of the analysis, we would like to ensure that the problem the algorithm solves in each step is always feasible (with probability 1). This is necessary for establishing that G holds with probability at least $1 - \delta$ (Theorem 1 of (Tranos et al., 2022)) as well as for our regret analysis.
