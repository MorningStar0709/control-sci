# 3.3. State Estimation for MPC

Sampling the future states $\mathbf { z } _ { t : t + K }$ in eq. (2) necessitates a distribution over the current state, the belief. This is done by a state estimator, which reads the observations into an internal state, or carry, which we denote with h. The carry is updated by a function $h ( \cdot , \cdot )$ which looks at the current observation and the previous carry and control: $\mathbf { h } _ { t + 1 } = h ( \mathbf { h } _ { t } , \mathbf { x } _ { t + 1 } , \mathbf { u } _ { t } )$ .

The form of the carry depends on the state estimator. For a particle filter, the carry would be the set of particles and weights. For an RNN, it would be the hidden state vector. Using the concept of the carry, we define the belief as $q ( \mathbf { z } _ { t } \mid \mathbf { h } _ { t } )$ , which acts as the distribution to use in eq. (2). It is often reasonable to assume that the initial state of the system is known $( \mathrm { e . g }$ . when we are only interested in relative motion). We formalise this as setting $\mathbf { h } _ { 1 } : = \mathbf { h } _ { \mathbf { z } } ^ { * }$ , where $\mathbf { h } _ { 1 }$ is the initial carry and $\mathbf { h } _ { \mathbf { z } } ^ { \ast }$ is a carry that deterministically identifies the true state z. We generally use the same notation z for both true and estimated states to keep notation simple.
