# II. PRELIMINARIES: CKKS CRYPTOSYSTEM

We briefly review the Cheon-Kim-Kim-Song (CKKS) cryptosystem [14], a fully HE scheme that enables approximate arithmetic directly on encrypted data.

The CKKS scheme includes algorithms KeyGen, Enc, and Dec. Given a security parameter $\lambda _ { \mathrm { s e c } } \in \mathbb { N }$ , the key generation algorithm KeyGen outputs a secret key sk. For a plaintext pt $\in \mathbb { C } ^ { N _ { \mathrm { c t } } / 2 }$ , where $N _ { \mathsf { c t } } ~ \in ~ \mathbb { N }$ is a powerof-two, the encryption algorithm takes sk and outputs a ciphertext $\mathsf { E n c } ( \mathsf { p t } ) \ \in \ C$ with C denoting the ciphertext space. The decryption algorithm uses sk to approximately recover the underlying plaintext, as $\mathsf { D e c } ( \mathsf { E n c } ( \mathsf { p t } ) ) \approx \mathsf { p t }$ . The approximation error arises from scaling and rounding during encoding and depends primarily on the scaling factor $\Delta > 0$ and the security parameter $N _ { \mathrm { c t } }$ (ring dimension).

The primitive homomorphic operations like addition, multiplication, and rotation are represented respectively by

$$\oplus : \mathcal {C} \times \mathcal {C} \to \mathcal {C}, \quad \otimes : \mathcal {C} \times \mathcal {C} \to \mathcal {C}, \quad \operatorname{RotCt} _ {\rho}: \mathcal {C} \to \mathcal {C}.$$
