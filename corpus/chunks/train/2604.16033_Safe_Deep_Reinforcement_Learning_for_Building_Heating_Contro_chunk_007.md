# 2.1 Optimization-Based Control

The control problem is formulated to determine optimal heat pump input ut ∈ R over the time horizon $\bar { t } \in \tau$ , subject to system dynamics and operational constraints:

$$\min _ {u _ {t}} \quad \mathcal {J} (u _ {t}, \zeta (t)) \quad , \tag {1a}$$

$\mathrm { s u b j e c t ~ t o } T _ { t } ^ { \mathrm { m i n } } \leq T _ { \mathrm { r o o m } , t } \leq T _ { t } ^ { \mathrm { m a x } } ,$ (1b)

$$T _ {\text { room }, t + 1} = h (T _ {\text { room }, t}, u _ {t}), \tag {1c}\sum_ {t \in \mathcal {H}} f (u _ {t}) \leq V (F), \tag {1d}$$

where $\mathcal { I }$ is the objective function, i.e. the electricity costs to be minimized, and $\zeta ( t )$ is the real-time electricity price at time t. We use $T _ { \mathrm { r o o m } , t }$ to denote the indoor room temperature at timestep t, and $T _ { t } ^ { \mathrm { m i n } }$ and $T _ { t } ^ { \mathrm { m a x } }$ are the minimum and maximum temperature limits for thermal comfort. The system dynamics $h ( T _ { \mathrm { r o o m } , t } , u _ { t } )$ capture the thermal behavior of the room, linking the current temperature and control input to the predicted room temperature at the next time step. This function depends on the building thermal model of the room. The term $f ( u _ { t } )$ represents the energy consumption associated with the control input, which is accumulated over the duration of a flexibility provision time window $t \in { \mathcal { H } } \subseteq \tau$ . The constraint in (1d) ensures that the total energy consumed within the flexibility time window does not exceed the allowable energy budget $V ( F )$ , as defined by the flexibility provision message F . This constraint introduces a global coupling across time steps, making the control problem temporally dependent for allocation of energy throughout the flexibility time window. More details and an explanation of flexibility constraints will be presented in Section 3.
