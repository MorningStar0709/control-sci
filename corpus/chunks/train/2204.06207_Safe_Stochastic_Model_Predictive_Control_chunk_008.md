# A. General Safe SMPC Algorithm

SMPC allows for a certain probability of constraint violation. Therefore, in order to use SMPC in a safe way, it needs to be ensured that applying an SMPC input is safe.

Fig. 1: Safe SMPC algorithm.

We propose a general safe SMPC algorithm that consists of an SMPC part and a backup predictive controller. This safe SMPC algorithm, shown in Figure 1, yields an input $\mathbf { \pmb { u } } _ { t }$ at each time step t, which is determined based on the following two modes:

• Stochastic mode (with $\mathrm { O C P } \mathbb { P } ^ { \mathrm { s } } ( { \pmb x } _ { t } ) )$   
• Backup mode (with OCP $\mathbb { P } ^ { \mathrm { b } } ( \pmb { x } _ { t } ) )$

We now present details on the two OCPs and on which mode to apply.

1) SMPC Optimal Control Problem: We consider the general SMPC OCP $\mathbb { P } ^ { \mathrm { s } } ( \pmb { x } _ { t } )$ with horizon N given by

$$\min _ {\boldsymbol {U} _ {t} ^ {s}} J (\boldsymbol {x} _ {t}, \boldsymbol {U} _ {t} ^ {s}) \tag {6a}\text { s.t. } \boldsymbol {x} _ {t + k + 1} = \boldsymbol {f} (\boldsymbol {x} _ {t + k | t}, \boldsymbol {u} _ {t + k | t} ^ {\mathrm{s}}, \boldsymbol {w} _ {t + k | t}) \tag {6b}\boldsymbol {u} _ {t + k \mid t} ^ {\mathrm{s}} \in \mathcal {U}, \quad k \in \mathbb {I} _ {0, N - 1} \tag {6c}\operatorname * {P r} \left(\boldsymbol {x} _ {t + k | t} \in \mathcal {X}\right) \geq \beta , \quad k \in \mathbb {I} _ {1, N} \tag {6d}$$

yielding the optimal input sequence $( \pmb { u } _ { t \vert t } ^ { s \ast } , . . . , \pmb { u } _ { t + N - 1 \vert t } ^ { s \ast } )$ A with the SMPC control law $U _ { t } ^ { \mathrm { s } * }$ = $\begin{array} { c c l } { \pmb { u } ^ { \mathrm { s } } ( \mathbf { \hat { x } } _ { t } ) } & { = } & { \pmb { u } _ { t \vert t } ^ { \mathrm { s } \ast } . } \end{array}$ Any SMPC method may be used to reformulate the chance constraint (6d) into a tractable formulation, depending on the uncertainty distribution.
