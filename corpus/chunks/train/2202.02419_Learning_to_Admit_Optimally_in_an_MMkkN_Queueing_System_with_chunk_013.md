Consider the queueing system sampled at arrival i for $i \in \{ 0 , 1 , \ldots \}$ , and let $A _ { i }$ denote the action of the dispatcher to admit or block arrival i. If arrival i is blocked, $A _ { i } = 0 ;$ ; otherwise, if arrival i is admitted (when there’s room), $A _ { i } = 1$ . We define $N _ { i }$ as the number of total jobs waiting for service in the server-queue pair before arrival $i ,$ and let $N _ { 0 } = 0$ . Let $T _ { i }$ be the inter-arrival time between arrival $i - 1$ and $i ,$ and $M _ { i }$ be the number of departures during $T _ { i }$ . Notice that $N _ { i - 1 } + A _ { i - 1 } = M _ { i } + N _ { i }$ and the value of $M _ { i }$ can be found with the knowledge of $\left\{ N _ { i - 1 } , N _ { i } , A _ { i - 1 } \right\}$ . The dispatcher chooses $A _ { i }$ based on past observations up to arrival i, i.e., $\mathcal { H } _ { i } = \left\{ T _ { 1 } , \dots , T _ { i } , A _ { 0 } , A _ { 1 } , \dots , A _ { i - 1 } , N _ { 0 } , N _ { 1 } , \dots , N _ { i } \right\}$ . Consider the policy class $\Pi = \{ \pi _ { a } , \pi _ { b } \}$ , where $\pi _ { b }$ is the policy that blocks all arrivals and $\pi _ { a }$ is the policy that accepts all arrivals, subject to available room. Using the history of observations, the dispatcher’s goal is to identify the optimal policy within the class Π that maximizes the expected average reward per unit time. We note that by PASTA (Srikant and Ying (2013)), the expected average reward per unit time is lim sup $\textstyle { \frac { 1 } { n } } \sum _ { i = 0 } ^ { n - 1 } \mathbb { E } [ K ( A _ { i } , \sigma _ { i } ) ]$ , where $\sigma _ { i }$ is the sojourn time of arrival i, and the reward function $K \left( \cdot , \cdot \right)$ is given by $K ( a , s ) = a ( R - c s )$ .
