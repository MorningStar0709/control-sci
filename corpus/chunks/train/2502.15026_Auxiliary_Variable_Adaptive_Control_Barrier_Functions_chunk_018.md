# B. Adaptive HOCBFs for Safety: AVCBFs

Motivated by the SACC example in Sec. IV-A, given a function $b : \mathbb { R } ^ { n } \ \to \ \mathbb { R }$ with relative degree m for system (1) and a time-varying vector $\mathbf { \boldsymbol { a } } ( t ) : = [ a _ { 1 } ( t ) , \ldots , a _ { m } ( t ) ] ^ { T }$ with positive components called auxiliary variables, the key idea in converting a regular HOCBF into an adaptive one without defining excessive constraints is to place one auxiliary variable in front of each function in (3) similar to (16). As described in Sec. IV-A, we only need to ensure each $a _ { i } ( t ) > 0 , i \in \{ 1 , . . . , m \}$ . However, if a function $b : \mathbb { R } ^ { n } $ R has a mixed relative degree for system (1) as seen in [16], i.e., not all control inputs will first appear in the same constraint after the same number of differentiations, simply multiplying each auxiliary variable with each function will not ensure that all components of u completely appear. To address this issue, we first propose the following definition:

Definition 6 (Minimum Relative Degree). If a function $b :$ $\mathbb { R } ^ { n } \to$ R has a mixed or high-order relative degree for system (1), the minimum relative degree, denoted by m, is the smallest number of times b needs to differentiate along dynamics (1) until at least one component of u explicitly shows.
