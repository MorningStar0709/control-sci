# 4.3 Indicator variables

We also define two indicator variables, $\ell _ { i } ^ { m } \in \{ 0 , 1 \}$ and $b _ { i } ^ { m } \in \{ 0 , 1 \}$ . The variable $\ell _ { i } ^ { m }$ is used to indicate whether the most recently transmitted output value $y _ { i }$ of agent $A _ { i }$ has been received by agent $\mathcal { A } _ { m } \left( \ell _ { i } ^ { m ^ { \star } } = 0 \right)$ , or that it still has to be received by agent $\mathcal { A } _ { m } \ ( \ell _ { i } ^ { m ^ { \ast } } = 1 )$ . Since information received by agent $\breve { A _ { m } }$ is processed at the sampling times, we assume that information is buffered if it is received between sampling instances. The variable $b _ { i } ^ { m }$ indicates whether agent $A _ { m }$ will process $( { \mathrm { i . e . } }$ , update $\widehat { y } _ { i } ^ { m } )$ the most recently transmitted output value by $\mathcal { A } _ { i } ~ \left( \tilde { b _ { i } ^ { m } } ~ = ~ 1 \right)$ or that $\widehat { y } _ { i } ^ { m }$ will not be updated at its next sampling instance $( b _ { i } ^ { m } = 0 )$ . We distinguish between these two “events” to ensure that updates of ${ \widehat { y } } ^ { m }$ align with the sampling times of agent $A _ { m } ,$ b as described in (7).
