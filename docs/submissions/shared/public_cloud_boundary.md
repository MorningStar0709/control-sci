# ControlMind 云端与本地边界说明

> 本文件定位：说明 ControlMind 为什么采用“云端公开体验 + 本地完整执行”的双形态交付。云端展示不是完整系统的公网化，而是完整系统在公开、脱敏、可回放场景下的交互切面。

## 云端 Demo 包含什么

云端 Demo 只处理公开或脱敏输入。典型输入包括公开 PDF URL、脱敏摘要、公开任务目标、预置医学 RAG trace 和报告中已经固化的样例。

云端 Demo 使用服务端代理。MinerU 官方 API、DeepSeek API 等密钥只由服务端读取，浏览器不接触 API Key，也不显示用户可复用的密钥形态。

云端 Demo 同时包含实时动作和 trace 回放。公开 PDF 解析、短文本出题评分、公开任务规划可以实时体验；批量评测、长任务飞轮、私有索引问答通过已验证 trace 回放和来源矩阵核验。

## 完整系统包含什么

完整系统处理私有资产、长任务、索引、模型和批量实验。包括原始 PDF、Markdown、图片、chunk、embedding、FAISS/BM25 索引、微调样本、模型权重、Judge 日志、批量排行榜和飞轮长任务。

完整系统默认本地优先。只有当数据被明确判定为公开或脱敏、且用户显式选择云端解析后端或兜底路径时，才允许调用云端服务。这个策略适用于控制科学、制造业、航天军工、医学等所有需要保留机构知识资产的场景，不把“医学隐私”作为唯一理由。

## 三条不可混淆的边界

| 边界 | 云端 Demo | 完整系统 |
|:---|:---|:---|
| 数据边界 | 公开/脱敏输入、预置样例、trace 摘要 | 私有原文、chunk、索引、模型权重、批量实验日志 |
| 执行边界 | 短 API、服务端代理、公开回放 | 长任务、资源调度、本地模型、私有索引、批量 Judge |
| 展示边界 | 展示方法论和可核验产物 | 展示生产能力、复现能力和私有部署能力 |

## 为什么这样设计

如果把完整系统直接搬到公网，既会暴露密钥和私有资产，也会把长任务稳定性、额度消耗和数据合规风险推给演示环境。如果云端只做静态页面，又无法体现系统具备可用能力。

因此 ControlMind 采用折中但可审计的设计：云端保留公开实时能力，重资产能力以 trace、DATA-TRACE、CLI 和报告附录核验。这样既保留云端部署能力的可见性，又不会破坏“默认本地，非必要不上云”的主叙事。

## 边界结论

云端等于公开/脱敏输入、服务端代理、trace 回放。

本地或完整系统等于私有资产、长任务、索引/模型、批量实验。

二者不是两套产品，而是同一套方法论在公开体验和私有部署两种场景下的边界切换。
